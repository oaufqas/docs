**Data Sources** (Источники данных) — это «мосты» между вашими данными и визуализацией Grafana. Как мы уже выяснили, Grafana сама ничего не хранит; чтобы она «ожила», ей нужно указать, откуда черпать информацию.

### Как работает Data Source

В настройках Grafana необходимо подключение, указывая:

- **URL:** Адрес, где живет ваша база данных (например, `http://prometheus:9090`).
- **Auth:** Логин/пароль или токены доступа.
- **Access:** Как Grafana должна обращаться к базе (через браузер пользователя или напрямую со своего сервера — почти всегда выбирают Server/Proxy).

#### Типы источников данных (grafana уникальна тем, что может объединять совершенно разные типы данных на одном экране):

А. Метрики (Time Series) это данные, которые меняются во времени (числа).

- **Prometheus:** Самый частый выбор. Использует язык **PromQL**.
- **VictoriaMetrics:** Совместима с Prometheus, но часто быстрее и компактнее.
- **InfluxDB:** Старая школа мониторинга систем и IoT.

Б. Логи (Logs) текстовые данные о событиях.

- **Loki:** «Prometheus для логов». Оптимизирован для работы в связке с метриками.
- **Elasticsearch:** Мощный поиск по огромным массивам текста.

В. Трассировки (Traces) пути запросов в микросервисах.

- **Tempo** и **Jaeger**.

Г. Реляционные базы данных (SQL) иногда бизнес-данные лежат в обычных таблицах.

- **PostgreSQL, MySQL, Microsoft SQL Server:** Grafana позволяет писать обычные SQL-запросы и превращать их результаты в графики.

Д. Облачные провайдеры

- **CloudWatch** (AWS), **Azure Monitor**, **Google Stackdriver**: Можно мониторить облака напрямую без установки экспортеров.

### Mixed Data Sources (Смешивание)

Это «киллер-фича» Grafana. На одной панели (графике) вы можете вывести:

1. Данные из **Prometheus** (загрузка CPU).
2. Данные из **Loki** (количество ошибок в логах за этот же период).  
    Это позволяет сразу увидеть корреляцию: «Ага, CPU подскочил ровно в тот момент, когда в логах посыпались ошибки БД».
## Настройка «Data Source as Code»

Как и дашборды, в нормальном продакшене источники данных не добавляют через меню. Их описывают в **YAML-файлах** в папке `provisioning`.

Пример конфига для Prometheus:

```yaml
apiVersion: 1
datasources:
  - name: Prometheus
    type: prometheus
    url: http://localhost:9090
    isDefault: true
```

Если вы положите такой файл в контейнер с Grafana, она при старте уже будет «знать» про ваш Prometheus.
