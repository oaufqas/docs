**Grafana** — это «лицо» всей системы наблюдаемости. Если Prometheus — это база данных, то Grafana — это интерактивное полотно, которое превращает сухие цифры в понятные графики, карты и таблицы.

### Как устроена Grafana

В отличие от многих инструментов, Grafana сама **не хранит данные**. Она работает как «умный посредник»:

1. Вы подключаете её к **[[datasources|Data Source]]** (Prometheus, PostgreSQL, Elasticsearch и т.д.).
2. Пишете запрос (на PromQL или SQL) прямо в интерфейсе Grafana.
3. Она отправляет этот запрос в базу, получает цифры и рисует график.

### Главные возможности

- **Универсальность:** Можно на одном дашборде вывести график из Prometheus (метрики) и таблицу из Elasticsearch (логи). Это и есть реализация концепции **MELT**.
- **[[dashboards|Динамические дашборды (Variables)]]:** Можно создать одну переменную `$host` и переключаться между серверами в выпадающем списке — все графики на странице мгновенно обновятся.
- **Аннотации:** Grafana может помечать на графиках важные события (например, вертикальная черта «Здесь был деплой новой версии»), чтобы вы видели, как это повлияло на нагрузку.
- **[[alerts|Alerting]]:** В современных версиях Grafana есть мощная система алертинга, которая может слать уведомления в Telegram/Slack на основе того, что она видит на графиках.
- Библиотека готовых дашбордов, ты просто копируешь **ID** (например, `1860` для Node Exporter), вставляешь его в свою Grafana, получаешь профессиональный мониторинг за 10 секунд.


Запуск в docker container:

```bash
docker run -d --name grafana (--network observability-net) -e GF_SECURITY_ADMIN_USER=admin -e GF_SECURITY_ADMIN_PASSWORD=admin -v grafana-storage:/var/lib/grafana -p 3000:3000 grafana/grafana-oss:latest
```
