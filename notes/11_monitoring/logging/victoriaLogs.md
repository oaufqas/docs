**VictoriaLogs** — это современная, сверхбыстрая и легковесная база данных для сбора и анализа логов, разработанная командой популярной тайм-серии VictoriaMetrics разработчикам яндекса. Написан на GO.

- Работает по push модели.
- Data Ingestion - Сохранение входящих логов (парсинг)
- Обогощение meta-данными (откуда пришли логи и тд.)

Прямое сравнение с Elasticsearch (ELK) и Grafana Loki показывает ключевые архитектурные преимущества:

| Критерий                 | Elasticsearch (ELK)                               | Grafana Loki                                                      | VictoriaLogs                                                                 |
| ------------------------ | ------------------------------------------------- | ----------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| **Ресурсы (CPU/RAM)**    | Очень прожорливый (Java GC)                       | Средний, но «душится» при тяжелых запросах                        | **Крайне низкое** (написан на Go, ест на 70-80% меньше ресурсов)             |
| **Место на диске**       | Огромное (из-за тяжелых инвертированных индексов) | Маленькое (индексирует только лейблы)                             | **Минимальное** (сжатие до 30-40 раз лучше, чем у конкурентов)               |
| **Поиск по тексту лога** | Быстрый, но дорогой                               | Медленный (делает полный перебор строк, если не угадал с лейблом) | **Молниеносный** (использует умные Bloom-фильтры для полнотекстового поиска) |
| **Сложность настройки**  | Высокая (индексы, шарды, маппинги)                | Высокая (настройка чанков, компакторов, s3-бэкендов)              | **Нулевая** (запускается одним бинарем, работает «из коробки»)               |
Основные фишки инструмента

1. **Концепция Schemaless (Без схем):** Ей всё равно, в каком формате приложение шлет логи (сырой текст, JSON, Stacktrace). Она автоматически индексирует **все** поля на лету.
2. **Язык запросов LogsQL:** У VictoriaLogs свой лаконичный язык запросов, основанный на пайпах (символ `|`), очень похожий на консольный `grep`.
    - _Пример:_ `{app="rent"} "database error" | stats count() by status_code`
3. **Встроенный Web UI:** В отличие от Loki, которому обязательно нужна Grafana, у VictoriaLogs есть свой минималистичный веб-интерфейс прямо «на борту» (порт 9428). Но интеграция с Grafana через плагин также поддерживается.
4. **Совместимость по API:** Она умеет «притворяться» Elasticsearch. Это значит, что любые старые агенты сбора логов (Logstash, Fluent Bit) могут слать логи в VictoriaLogs без изменения своего кода

Пример манифеста argocd application для kubernetes:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: victorialogs-stack
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.io'
    chart: victoria-logs-single # Одиночный инстанс (мало ест ресурсов)
    targetRevision: 0.7.0 # Укажи актуальную версию
    helm:
      values: |
        server:
          persistence:
            enabled: true
            storageClass: "yc-network-hdd" # Твой диск в Яндексе
            size: 10Gi
          retentionPeriod: "7d" # Хранить логи ровно 7 дней для экономии денег
        
        # Включаем встроенный сборщик логов Fluent Bit или Vector
        vector:
          enabled: true
  destination:
    server: 'https://default.svc'
    namespace: logging
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    createNamespace: true
```

Пример фильтрации и поиска логов

LogsQL спроектирован так, чтобы быть проще и быстрее, чем LogQL (у Loki) и KQL (у Elastic). Он ищет данные через простые логические выражения:

- **Найти все логи вашего сервера:**  
    `_stream:{app="rent"}`
- **Полнотекстовый поиск ошибки (молниеносный, благодаря Bloom-фильтрам):**  
    `_stream:{app="rent"} AND "Database connection failed"`
- **Поиск по точному совпадению или части слова:**  
    `_stream:{app="rent"} AND error*`
- **Поиск по конкретному полю (если сервер пишет логи в JSON):**  
    `_stream:{app="rent"} status_code:500`
