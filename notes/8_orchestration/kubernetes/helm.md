**Helm** — это пакетный менеджер для Kubernetes (аналог `npm` для Node.js или `apt` для Ubuntu).

Если обычный манифест — это «чертеж одного кирпича», то **Helm-чарт** — это «чертеж всего здания», где ты можешь менять количество этажей или цвет стен одной переменной.

Зачем он нужен?

1. **Шаблонизация (Template):** В обычном YAML нельзя использовать переменные. В Helm можно написать `{{ .Values.image }}` и подставлять разные образы для Dev и Prod сред, не плодя копии файлов.
2. **Управление как единым целым:** Ты можешь поставить сразу Deployment + Service + Ingress + HPA одной командой `helm install`. И так же одной командой всё удалить.
3. **Версионность и Откаты:** Helm помнит историю всех твоих «релизов». Если ты обновился и всё сломалось, команда `helm rollback` за секунду вернет предыдущую рабочую конфигурацию. Вся информация о релизах хранится в секретах default namespace. Всего может хранится 10 ревизий (можно законфигурировать).

---

Из чего состоит Helm-чарт? (Структура папки)

Когда ты создаешь чарт (`helm create my-app`), он генерирует такую структуру:

- **`Chart.yaml`** — паспорт проекта (название, версия приложения, версия чарта).
- **`values.yaml`** — **самый важный файл**. Здесь лежат все настройки (порт, количество реплик, название образа). Это то, что правит человек.
- **`templates/`** — папка с «дырявыми» YAML-манифестами. Вместо конкретных значений там стоят заглушки вроде `{{ .Values.replicas }}`.
- **`charts/`** — папка для зависимостей (например, если твоему приложению нужен Redis, его чарт можно положить сюда).

---

Как это работает на примере

В файле `values.yaml` ты пишешь:

```yaml
replicaCount: 5
image: rvlxx/js:1.0
```

А в файле `templates/deployment.yaml` у тебя:

```yaml
spec:
  replicas: {{ .Values.replicaCount }}
  template:
    spec:
      containers:
        - name: app
          image: {{ .Values.image }}
```

Когда ты запускаешь установку, Helm «склеивает» эти файлы в готовый манифест и отправляет его в Kubernetes.

---

Основные команды:

- **`helm create my-chart`** — создать заготовку нового чарта.
- **`helm install my-release ./my-chart`** — развернуть всё приложение в кластер под именем `my-release`.
- **`helm list`** — список всех установленных приложений.
- `helm install --debug --dry-run rent ./my-app-chart` — это не будет ничего устанавливать, а просто выведет в терминал финальный YAML.
- **`helm upgrade my-release ./my-chart`** — обновить приложение после того, как ты что-то поменял в `values.yaml`.
- **`helm rollback my-release 1`** — откатиться к первой версии релиза.
- **`helm repo add bitnami ...`** — подключить внешние репозитории (магазины готовых приложений). Можно поставить себе готовый Postgres или Grafana одной командой.
- `helm upgrade app . --set ssl.enabled=false` — перезапустить чарт с переопределенной переменной
- `--atomic` — helm автоматически сделает rollback новой версии к работающей старой
- `helm install app -f values-dev.yaml` — явно указать файл переменных при запуске
- `helm lint app` — проверка на ошибки в манифестах, используется в ci-cd
- `helm package app` — запаковать чарт в архив

### Подробный синтаксис и примеры

#### Вывод значений (Interpolation)

Самое простое — вставка данных. Все команды пишутся внутри двойных фигурных скобок `{{ }}`.

- `{{ .Values.server.port }}` — берет значение из `values.yaml`.
- `{{ .Release.Name }}` — встроенная переменная (имя релиза).
- `{{ .Chart.Name }}` — имя твоего чарта из `Chart.yaml`.

#### Функции и Пайпы (Pipes)

Helm позволяет модифицировать строки на лету с помощью «пайпов» (символ `|`), как в Linux.

- `{{ .Values.domain | upper }}` — переведет домен в верхний регистр.
- `{{ .Values.domain | quote }}` — обернет значение в кавычки (очень важно для строк).
- `{{ .Values.domain | default "rent.local" }}` — если значение в `values.yaml` пустое, подставит дефолт.

#### Управляющие конструкции (Logic)

**If / Else (Условия)** — позволяет включать или выключать блоки манифеста.

```yaml
{{- if .Values.ingress.enabled }}
apiVersion: networking.k8s.io/v1
kind: Ingress
...
{{- else }}
# Ingress disabled
{{- end }}
```

- **Тильда `{{-`**: удаляет лишние пробелы и пустые строки вокруг блока. Без неё в итоговом YAML будут дырки.

**With (Смена области видимости)** — чтобы не писать каждый раз `.Values.server.container.something`, можно "войти" внутрь объекта:

```yaml
{{- with .Values.server.container }}
image: {{ .image }}:{{ .tag }} # Теперь мы пишем относительно container
port: {{ .port }}
{{- end }}
```

**Range (Циклы)** — Используется для создания списков (например, env-переменных).

```yaml
env:
{{- range .Values.envList }}
  - name: {{ .name }}
    value: {{ .value | quote }}
{{- end }}
```

 **Использование "Именованных шаблонов" (Helpers)** — в папке `templates/` часто лежит файл `_helpers.tpl`. Там описываются куски кода, которые повторяются (например, стандартные лейблы).

- **Определение:** `{{- define "mychart.labels" -}} ... {{- end -}}`
- **Вызов:** `{{ include "mychart.labels" . }}`


**Работа с пробелами (Indentation)** — В YAML пробелы — это всё. В Helm есть функция `nindent` (New Line + Indent), которая сдвигает блок кода на нужное количество пробелов.

```yaml
metadata:
  labels:
    {{- include "my-app.labels" . | nindent 4 }}
```

#### Встроенные объекты

Кроме `.Values`, у тебя всегда под рукой:

- `.Release`: `.Release.Namespace`, `.Release.IsInstall`, `.Release.IsUpgrade`.
- `.Capabilities`: информация о версии Kubernetes (чтобы понимать, поддерживается ли Ingress V1).
- `.Files`: позволяет прочитать содержимое любого файла в чарте (например, `{{ .Files.Get "config.json" }}`).
