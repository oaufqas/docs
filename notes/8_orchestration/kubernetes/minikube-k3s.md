**Minikube** — это инструмент, который создает локальный одноузловой кластер Kubernetes на вашем компьютере. Он предназначен для обучения, экспериментов и тестирования приложений в среде, максимально приближенной к «боевому» Kubernetes. 
#### Как это устроено:
Minikube запускает Kubernetes внутри изолированной среды, используя **драйверы** для связи с вашей операционной системой: 

1. **Среда выполнения (Runtime):** Minikube разворачивает виртуальную машину или контейнер, которые имитируют физический сервер (узел).
2. **Драйверы изоляции:** В зависимости от ОС и настроек, Minikube может работать через:
	- **Контейнерные драйверы (Docker, Podman):** Minikube запускает один большой «системный» контейнер. Внутри него работают свои собственные контейнерные движки (Docker-in-Docker или Containerd) и все процессы K8s.
	- **Виртуализация (VirtualBox, KVM, Hyper-V):** Создается минималистичная виртуальная машина на базе специально подготовленного образа **Minikube ISO** (на основе Buildroot).
    - **Bare-metal:** Запуск компонентов напрямую в Linux (требует прав root).
3. **Компоненты кластера:** Внутри созданной среды запускаются все стандартные процессы Kubernetes: API-сервер, планировщик (scheduler), контроллеры и агент узла (kubelet).
### Install minikube

 1. Install (Linux x86-64)

```bash
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube
```

2. Start

```bash
minikube start
```


Популярные команды Minikube

1. **`minikube start/stop`** — Магия создания. Находит образ узла, запускает главный контейнер, пробрасывает порты и генерирует сертификаты для `kubectl`.
2. **`minikube status`** — Проверка «здоровья». Проверяет, запущен ли главный контейнер и отвечают ли внутри него компоненты (apiserver, kubelet).
3. **`minikube delete`** — Удаляет контейнер с кластером
4. **`minikube dashboard`** — Запускает веб-интерфейс. Команда поднимает прокси-сервер между твоим браузером и системным подом `kubernetes-dashboard` внутри кластера.
5. **`minikube ip`** — Выводит внутренний IP-адрес главного контейнера в сети Docker.
6. **`minikube ssh`** — Пробрасывает терминал внутрь главного контейнера. Это как `docker exec -it minikube bash`, только с настроенным окружением.
7. **`minikube service <name>`** — Самая полезная команда для новичка. Она узнает, на каком порту работает твое приложение внутри, и открывает его в браузере хоста.
8. **`minikube addons enable <name>`** — Включает плагины (например, `ingress` или `metrics-server`). Minikube просто закидывает нужный YAML-файл в папку автозапуска внутри узла.
9. **`minikube pause / unpause`** — Ставит на паузу все процессы в контейнерах, чтобы они не ели CPU, пока ты пьешь кофе.
10. **`minikube version`** — Показывает версию minikube
11. **`minikube start --cpus=4 --memory=8000mb --disk-size=5gb`** — Запуск с параметрами

### K3S - полноценная дистрибуция Kubernetes

1. **Легковесное хранилище (замена `etcd`)**.
	- Стандартный Kubernetes и Minikube для хранения состояния кластера используют базу данных **etcd**.
	- K3s по умолчанию использует гораздо более легкую встроенную базу данных **SQLite**. Благодаря этому кластер стартует значительно быстрее и потребляет меньше памяти.
2. **Упаковка в один бинарный файл.**
    - K3s упакован в один бинарный файл размером **менее 100 МБ**, который включает в себя всё необходимое для работы: `kubectl`, `containerd` (для запуска контейнеров), `Flannel` (для сети), `CoreDNS`. За счёт этого пропадает множество зависимостей и ускоряется запуск.
3. **Простота установки.**
    - В то время как Minikube требует предустановленного гипервизора или Docker, K3s устанавливается **одной командой** прямо на хост-систему, работая как обычный сервис.


Базовая установка сервeра:

```bash
curl -sfL https://get.k3s.io | sh -
# INSTALL_K3S_EXEC="--disable traefik --disable servicelb" sh - Без Traefik
```

Настройка контекста для kubectl:

```bash
# Создаем папку для конфига, если её нет
mkdir -p ~/.kube

# Копируем конфиг K3s в стандартное место
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config

# Меням владельца файла на твоего текущего пользователя
sudo chown $USER:$USER ~/.kube/config

# Проверяем работу кластера
kubectl get nodes
```

Чтобы получить токен для подключения рабочих нод:

```bash
sudo cat /var/lib/rancher/k3s/server/node-token
```

Подключение рабочей ноды:

```bash
curl -sfL https://get.k3s.io | K3S_URL=https://<АДРЕС_ВАШЕГО_СЕРВЕРА>:6443 K3S_TOKEN=<ТОКЕН_ИЗ_СЕРВЕРА> sh -
```

Удаление K3s

- **Удаление с сервера:** `/usr/local/bin/k3s-uninstall.sh`
- **Удаление с агента:** `/usr/local/bin/k3s-agent-uninstall.sh`
