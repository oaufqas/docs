### Файловые системы 

- **Ext4 (Fourth Extended Filesystem)** - Стандарт по умолчанию для большинства дистрибутивов Linux. Очень стабильная, проверенная временем и быстрая файловая система. Поддерживает огромные файлы и диски, использует журналирование (защиту от сбоев питания).
- **Btrfs (Butter FS)** - Современная «умная» система. Главные фишки — создание мгновенных снимков системы (снапшотов), встроенный RAID (объединение дисков) и сжатие данных «на лету» для экономии места. Используется по умолчанию в Fedora и openSUSE.
- **XFS** - Высокопроизводительная файловая система, которая создавалась для работы с огромными массивами данных. Отлично справляется с очень большими файлами и тяжелыми нагрузками. Используется по умолчанию в Red Hat Enterprise Linux (RHEL).
- **ZFS** - Гибрид файловой системы и менеджера логических дисков. Обладает невероятной надежностью, защитой от повреждения данных, продвинутым сжатием и снапшотами. Идеальна для серверов хранения данных (NAS), но требует много оперативной памяти.
- **NTFS (New Technology Filesystem)** - Основная файловая система для Windows (диск C:, D: и т.д.). Поддерживает права доступа, шифрование, сжатие и большие файлы. Linux умеет читать и писать на NTFS, но «родной» для него она не является.
- **ReFS (Resilient Filesystem)** - Новое поколение от Microsoft для серверов и корпоративных хранилищ. Заточена под максимальную отказоустойчивость, автоматическое исправление поврежденных файлов и работу с огромными объемами данных.
- **APFS (Apple File System)** - Современный стандарт для Mac, оптимизированный специально под SSD и Flash-накопители. Поддерживает быстрое шифрование, мгновенное копирование файлов (клонирование) и динамическое распределение свободного места.
- **HFS+ (Mac OS Extended)** - Старая файловая система Apple. Сейчас используется в основном для совместимости со старыми компьютерами Mac или для резервных копий Time Machine на старых внешних жестких дисках.
- **FAT32** - Очень старый стандарт, который **понимают абсолютно все устройства**: Linux, Windows, macOS, телевизоры, магнитолы и консоли. Главный минус — жесткое ограничение: нельзя записать один файл размером **больше 4 ГБ**.
- **exFAT** - Современная замена FAT32, созданная специально для флешек и внешних дисков. У нее нет ограничений на размер файла (можно спокойно писать фильмы по 50 ГБ), и она официально поддерживается и в Windows, и в macOS, и в Linux.

### Разделы дисков для загрузки ОС

Системный раздел EFI (ESP — EFI System Partition)

Это стандарт для всех современных компьютеров, работающих в режиме **UEFI** (вытеснившем старый BIOS). Он используется в Windows, Linux и macOS.

- **Файловая система:** Строго **FAT32** (изредка FAT16/FAT12). Это требование зашито прямо в материнскую плату. Микропрограмма UEFI умеет читать файлы _только_ из разметки FAT, так как она очень простая и для нее не нужны сложные драйверы.
- **Что там лежит:** Файлы с расширением `.efi` (загрузчики ОС). Например:
    - Для Linux: загрузчик GRUB (`/EFI/BOOT/BOOTX64.EFI` или `/EFI/ubuntu/grubx64.efi`).
    - Для Windows: диспетчер загрузки (`/EFI/Microsoft/Boot/bootmgfw.efi`).
- **Размер:** Обычно небольшой, от 100 до 500 МБ.
- **Особенность:** Он имеет специальный флаг (флаг загрузки) в таблице разделов GPT, поэтому обычные операционные системы скрывают его от пользователя в проводнике, чтобы никто случайно ничего не удалил.

Разделы для старых систем (Legacy BIOS / MBR)

Если компьютер старый или настроен в режиме эмуляции BIOS, там нет файловой системы FAT32 для загрузки. Вместо этого используется классический подход:

- **MBR (Master Boot Record):** Самый первый сектор диска (512 байт). В нем нет никакой файловой системы. Там лежит крошечный кусок кода (boot sector), который указывает материнской плате, куда смотреть дальше.
- **В Windows (Раздел «Зарезервировано системой»):** 
	- Создается автоматически при установке Windows на MBR-диск. Файловая система там **NTFS**, размер около 100–500 МБ. Он скрыт, не имеет буквы диска и содержит базу данных конфигурации загрузки (BCD).
- **В Linux (Раздел `/boot` или BIOS Boot Partition):**
    - Если Linux ставится на GPT-диск, но в режиме BIOS, создается микро-раздел **BIOS Boot Partition** (размером всего 1–2 МБ) вообще без файловой системы — туда GRUB записывает свой базовый код.
    - Также часто создается отдельный раздел `/boot` с файловой системой **Ext4**. На нем лежат файлы ядра Linux (`vmlinuz`) и виртуальный диск инициализации (`initrd`). Он может быть как скрытым, так и видимым в системе, но монтируется отдельно.


### Стандарт POSIX (Portable Operating System Interface for Unix)

**POSIX — это абстрактный стандарт**, а не программа, не код и не операционная система. Это просто огромный текстовый документ (спецификация), в котором написано: _«Если вы хотите, чтобы ваша ОС считалась правильной и совместимой, она должна делать вот это, вот так и отвечать на такие-то команды»_.

Можно сказать это специальный "язык" запросов и ответов к ядру, для выполнения пользовательских программ. 
Программа --> Ядро --> Оборудование --> Необходимые ресурсы для программы

- Каждая ОС имеет свой интерфейс взаимодействия с ядром, а POSIX как стандарт, его можно придерживаться, тогда ОС будет считаться POSIX-совместимой.
- Обеспечивает совместимость UNIX-подобных операционных систем

Стандарты POSIX описывают:
- Системный интерфейс (Вызовы ядра)
- Командный интерпретатор
- Утилиты

POSIX - это работа в пространстве пользователя, системные вызовы работают в пространстве пользователя

## Типы файлов в Linux

| Тип                     | Обозначение | Пример     |
| ----------------------- | :---------: | ---------- |
| Обычный файл            |      -      | -rw-r--r-- |
| Директория              |      d      | drwxr-xr-x |
| Символическая ссылка    |      l      | lrwxrwxrwx |
| Устройство (блочное)    |      b      | brw-rw---- |
| Устройство (символьное) |      c      | crw-rw---- |
| Сокет                   |      s      | srwxrwxrwx |
| Pipe (FIFO)             |      p      | prw-r--r-- |


Основные команды

| Команда | Описание                 | Пример                   |
| ------- | ------------------------ | ------------------------ |
| df      | Использование дисков     | df -h (человекочитаемый) |
| du      | Размер директорий        | du -sh * (суммарно)      |
| mount   | Монтирование             | mount /dev/sda1 /mnt     |
| umount  | Размонтирование          | umount /mnt              |
| fdisk   | Работа с разделами       | fdisk -l (список)        |
| lsblk   | Список блочных устройств | lsblk                    |
| blkid   | UUID устройств           | blkid                    |
| fsck    | Проверка ФС              | fsck /dev/sda1           |
| sync    | Сброс кэша на диск       | sync                     |

#### Сжатие файлов:

`tar -cf archive.tar dir/`     # Создать tar
`tar -xf archive.tar`          # Распаковать tar

`tar -czf archive.tar.gz dir/`     # Создать tar.gz
`tar -xzf archive.tar.gz`          # Распаковать tar.gz

`tar -cjf archive.tar.bz2 dir/`    # Создать tar.bz2
`tar -xjf archive.tar.bz2`          # Распаковать tar.bz2

`zip -r archive.zip dir/`          # Создать zip
`unzip archive.zip`                # Распаковать zip

`gzip file`                        # Сжать файл
`gunzip file.gz`                   # Распаковать gz

### Файловая система

```bash
FHS (Filesystem Hierarchy Standard)

/ (root)
├── bin  → /usr/bin        # Базовые команды (ls, cp, mv)
├── boot                   # Загрузчик, ядро
├── dev                    # Устройства (sda, tty, null)
├── etc                    # Конфигурационные файлы
├── home                   # Домашние папки пользователей
├── lib  → /usr/lib        # Системные библиотеки
├── media                  # Точки монтирования сменных носителей
├── mnt                    # Временные точки монтирования
├── opt                    # Дополнительное ПО
├── proc                   # Виртуальная ФС с информацией о процессах
├── root                   # Домашняя папка root
├── run                    # Данные о запущенных процессах
├── sbin → /usr/sbin       # Системные команды
├── srv                    # Данные сервисов
├── sys                    # Виртуальная ФС с информацией о системе
├── tmp                    # Временные файлы
├── usr                    # Пользовательские программы
│   ├── bin
│   ├── lib
│   └── local              # ПО, установленное вручную
└── var                    # Изменяемые данные (логи, кэш)
    ├── log
    ├── cache
    └── lib
```
