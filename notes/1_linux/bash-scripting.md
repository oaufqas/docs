**Любая команда, которую вы вводите в терминале, может быть записана в скрипт**. Для исполнения файлов, сначала нужно выдать право на это `chmod +x <filename>`
### Анатомия скрипта и первая строчка (Shebang)

Каждый Bash-скрипт должен начинаться со строчки **Shebang** (`#!`). Она указывает операционной системе, какой интерпретатор использовать для запуска этого файла.

- **Правильный вариант:** `#!/usr/bin/env bash`
    - _Почему:_ Этот вариант находит `bash` в системе независимо от того, куда его установил администратор (`/bin/bash` или `/usr/local/bin/bash`). При создании юнит сервисов для скрипта, **шебанг обязателен**.

### Переменные и кавычки

- **Без пробелов:** При объявлении переменных пробелы вокруг знака `=` ставить **нельзя**: `NAME="ArgoCD"` (запись `NAME = "ArgoCD"` выдаст ошибку).
- **Регистр:** Переменные окружения принято писать капсом (`PATH`, `USER`), а внутренние переменные скрипта — маленькими буквами (`my_var`), чтобы случайно не перезаписать системные.
- Если не объявить переменную, но использовать ее в скрипте, ошибки не будет, она будет считаться нулем, но в таком состоянии выводиться не будет, но если например сделать `echo $(($n+1))`, вывод будет 1.

- **Двойные кавычки (`"`)** — разрешают подстановку переменных внутри строки.
- **Одинарные кавычки (`'`)** — экранируют всё, строка воспринимается буквально как текст.
- **Всегда берите переменные в кавычки**, когда используете их, иначе пробелы внутри значения «сломают» команду.



```bash
#!/usr/bin/env bash

tool="Kubernetes"

echo "I love $tool"  # Выведет: I love Kubernetes
echo 'I love $tool'  # Выведет: I love $tool

# Хорошая привычка (всегда в кавычках):
echo "${tool}"
```

#### Массивы

В bash, массивы могут иметь элементы разных типов данных, объявление массива:

```bash
my_arr=(6 123 4 15 1 23 "str" 3)
echo ${my_arr[@]} # Вывести все элементы
# 6 123 4 15 1 23 "str" 3
echo ${!my_arr[@]} # Вывести все индексы
# 1 2 3 4 5 6 7
echo ${#my_arr[@]} # Общее колличество элементов, в этом случае индексов
# 7
```

##### Специальные переменные (Аргументы)

Когда вы запускаете скрипт `./deploy.sh prod app-v2`, вы передаете ему аргументы. Внутри скрипта они доступны через специальные переменные:

- `$0` — Имя самого скрипта (например, `./deploy.sh`).
- `$1`, `$2`, `$3`... — Первый, второй, третий аргументы (в примере `$1` станет `prod`, а `$2` — `app-v2`). После 9 параметров: `$[10]`
- `$#` — Количество переданных аргументов.
- `$@` — Все аргументы одной строкой (удобно для перебора в цикле).
- `$?` — **Самая важная переменная!** Код возврата последней выполненной команды. `0` означает успех, любое другое число (от 1 до 255) — ошибку.

##### Переменные окружения

- BASH — полный путь до исполняемого файла bash
- BASH_VERSION — версия bash
- HOME — домашний каталог пользователя, который запустил скрипт
- RANDOM — случайное число в диапазоне от 0 до 32.767
- OSTYPE — тип операционной системы
- PWD — текущий каталог
- PS1 — строка приглашения
- UID — id пользователя, запустившего сценарий
- USER — имя пользователя

### Условия и Магия конструкции `[[ ... ]]`

![[Pasted image 20260616205603.png]]

Для проверки условий в современном Bash используется конструкция `[[ ... ]]`. Внутри нее используются свои флаги для проверки файлов и строк.

Проверка файлов:

- `-f $file` — Истинно, если файл существует и это обычный файл.
- `-d $dir` — Истинно, если директория существует.
- `-e $path` — Истинно, если путь вообще существует (неважно, файл или папка).

Сравнение строк:

- `-z $str` — Строка пустая (длина равна 0).
- `-n $str` — Строка НЕ пустая.
- `==` и `!=` — Равно и не равно.

Сравнение чисел (Здесь используются буквы!):

- `-eq` (Equal) — Равно (`[[ $a -eq $b ]]`)
- `-ne` (Not Equal) — Не равно
- `-gt` (Greater Than) — Больше
- `-lt` (Less Than) — Меньше
- `-le` (Less Equal) — меньше или равно
- `-ge` (Greater Equal) — больше или равно


```bash
#!/usr/bin/env bash

# Пример проверки аргумента
echo -n "enter your age: "
read AGE
if [[ "$AGE" -ge 0 ]] && [[ $AGE -lt 12 ]]; then
        echo "child"
    elif [[ $AGE -ge 12 ]] && [[ $AGE -lt 18 ]]; then
        echo "teenager"
    elif [[ $AGE -ge 18 ]] && [[ $AGE -lt 60 ]]; then
        echo "man"
    else
        echo "old man"
fi

  
echo -n "enter your telephone brand: "
read brand  

if [[ $brand == "samsung" ]] || [[ $brand == "nokia" ]] || [[ $brand == "huawei" ]] || [[ $brand == "iphone" ]]; then
    case $brand in
        samsung)
            echo "-10%";;
        nokia)
            echo "-20%";;
        huawei)
            echo "-30%";;
        *)
            echo "else variant"
    esac
else
    echo "$brand - not found in discount list"
fi
```

### Циклы и подстановка команд

Чтобы записать вывод какой-то Linux-команды в переменную, используйте синтаксис `$(команда)`.


```bash
#!/usr/bin/env bash

# Записываем текущую дату в переменную
current_date=$(date +%Y-%m-%d)
echo "Today: $current_date"

# Цикл по списку элементов
for app in auth payment notification; do
    echo "Restart service: $app"
done

# Цикл по файлам в папке
for file in /etc/*.conf; do
    echo "Config-file found: $file"
done

for (( i=0; i<10; i++ )); do
	echo $i;
done
```

`>` - перезаписать содержимое файла
`>>` - добавить в конец файла 
`<` - вместо ожидания ввода, будет использованы данные слева

Если вы пишете скрипт, который будет запускаться в CI/CD (например, в GitLab CI), добавьте в самое начало (сразу после Shebang) следующую строчку настроек:


```bash
set -euo pipefail
```


**Что делает эта магия:**

- `-e` (Exit on error): Скрипт мгновенно завершит работу с ошибкой, если хоть одна команда упадет. По умолчанию Bash продолжит выполнять следующие строчки, даже если посреди скрипта всё сломалось.
- `-u` (Unset variables): Скрипт упадет, если вы попытаетесь использовать переменную, которую забыли объявить. Защищает от опечаток (например, если вместо `$KUBE_TOKEN` вы случайно написали `$KUB_TOKEN`).
- `-o pipefail`: Если внутри цепочки команд (пайпа) вида `cat file.txt | grep "error" | awk '{print $2}'` упадет первая команда `cat`, весь пайп признается упавшим. По умолчанию Bash смотрит код ответа только _последней_ команды (`awk`), маскируя ошибки в начале цепочки.


### Функции

Нужны для того, чтобы избегать дублирования кода, позволяют переиспользовать части кода, с разными параметрами, синтаксис объявления и использования функций:

```bash
func() {
	echo "h";
}

func    # вызов функции
```
