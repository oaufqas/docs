| Команда | Описание | Пример                  
|---|---|---
| `chmod`   | Изменить права       | chmod 755 script.sh 
| `chown`   | Изменить владельца   | chown user:group file
| `umask`   | Маска по умолчанию   | umask 022

| `usermod` | Изменить пользователя| usermod -aG docker user (Добавляет в группу)

| `groups`  | Группы пользователя  | groups user

### Права

В первом столбце записи -rw-rw-rw- отображается тип файла:
- - (прочерк) - обычный файл
- l - симлинк
- d - директория

Первые 3 значения rwx - права владельца, вторые - права группы, третьи - для всех осталных.

Изменять права можно командой chmod, несколькими способами:
`chmod 755 file.txt` - с помощью битов:
- 4 = read (r)
- 2 = write (w)
- 1 = execute (x)
`chmod o+r,u+w,g+x,+x file.txt` - с помощью букв, где:
- u - owner
- g - group
- o - other

`s` - бит передачи полномочий, например когда пользователь без рут прав меняет пароль своей учетной записи, меняется файл `/etc/shadow` с правами `-rw-r-----`, это возможно благодаря правам `-rwSr-xr-x` утилиты `passwd`, и поэтому утилита выполнялась от лица ее владельца (root). (SUID - set user id)

`t` - бит "липучка", закрепляет права за конкретным пользователем

**Примеры:**
- `chmod 755` = rwxr-xr-x (владелец всё, остальные чтение/выполнение)
- `chmod -R 755 <dir>` = rwxr-xr-x (Меняет права доступа всех дочерних директорий и файлов)
- `chmod 644` = rw-r--r-- (владелец чтение/запись, остальные только чтение)
- `chown newuser newfile` – Меняет владельца newfile на newuser
- `chown newuser:newgroup newfile` – Изменяет владельца и группу-владельца для newfile на newuser и newgroup
- `chown -R newuser:newgroup newfolder` – Меняет владельца и группу-владельца каталога newfolder на newuser и newgroup
- `stat -c “%U %G” newfile` – отображает владельцев пользователей и групп newfile

```bash
sudo echo "<username> ALL=(ALL:ALL) NOPASSWD:ALL" >> /etc/sudoers - права sudo без пароля

cat /etc/group - Вывести все группы
cat /etc/passwd - Вывести всех пользователей
cat /etc/shadow - Зашифрованные пароли пользователей
```

Менять разрешения на использование sudo: `etc/sudoers` или `sudo visudo`

### ACL - Access Control List


**Проблема UGO:** Эта модель плоская. Вы не можете дать права на файл двум разным группам или трем конкретным пользователям. Либо один владелец, либо одна группа, либо весь мир. Для решения этой проблемы и придумали **ACL** - POSIX Access Control List, позволяющий расширить таблицу разрешений для файлов, дополнительным пользователям и группам. Добивить их можно с помощью утилиты `setfacl`, например:

`setfacl -m u:ansible:r file.txt` - добавит права `read` у юзера `ansible` и теперь права файла будут отображаться как `-r--r--r--+` (плюс в конце), детально посмотреть на дополнительные права можно так `getfacl file.txt`

Ядро Linux проверяет права строго **сверху вниз** до первого совпадения и сразу прекращает проверку. Алгоритм для вашего файла выглядит так:

1. **Проверка на владельца (Owner):** Вы залогинены под пользователем `rvlx`. Ядро смотрит на строку владельца в ACL: `user::---`. Вывод ядра: _«Так, текущий пользователь — владелец, и у него права `---` (полный нуль)»_.
2. На этом проверка **немедленно завершается**. Ядро выдает отказ (`Permission denied`).

Ядро даже **не доходит** до анализа строк `group:rvlx:rwx` или `mask::rwx`, где у вас прописаны полные права. Логика POSIX ACL непреклонна: если вы являетесь владельцем файла, то действуют _только_ права владельца (`user::`), а групповые права для вас полностью игнорируются, даже если вы состоите в этой группе

**Маска (`mask::`)** — это **«верхний потолок» (максимальный ограничитель)** для всех именованных пользователей и групп в ACL. Она определяет максимальные права, которые они _в принципе_ могут получить, независимо от того, что написано в их персональных строках.

Для обратной совместимости утилита `chmod` применительно к группе (`g`) на файлах с ACL **изменяет не саму группу, а маску**

На кого маска влияет, а на кого нет:

- **Не влияет** на владельца файла (`user::`).
- **Не влияет** на категорию «остальные» (`other::`).
- **Влияет на всех остальных:** на именованных пользователей (`user:username`), на основную группу (`group::`) и на именованные группы (`group:groupname`)
