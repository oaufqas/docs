#### Что такое TTY (Терминал)

**TTY (Teletypewriter / Телетайп)** — это абстракция или файл устройства, который отвечает за **ввод и вывод текста**.

- **История:** В 1970-х годах компьютеры Unix не имели мониторов. Они были размером со шкаф, а пользователи сидели за телетайпами — гибридами печатной машинки и принтера. Вы нажимали механическую клавишу, буква летела по проводу в Unix, Unix обрабатывал её и присылал обратно команду напечатать ответ на бумаге.
- **Как это устроено сейчас:** Сегодня физических телетайпов нет, но концепция осталась. В Linux терминал — это программная эмуляция того провода. Он бывает двух видов:
    1. **Физический TTY (`/dev/tty1` – `/dev/tty6`):** Если на сервере Linux нажать `Ctrl+Alt+F1`, графическая оболочка скроется, и перед вами появится чистое черное окно ввода текста. Это полноэкранный терминал, зашитый прямо в ядро Linux.
    2. **Псевдо-терминал (PTS — Pseudo-Terminal Slave, `/dev/pts/0...`):** Когда вы запускаете приложение «Терминал» внутри графического интерфейса (Ubuntu, macOS) или подключаетесь по SSH, ядро создает виртуальное устройство. Это то самое устройство, в дескриптор которого программы пишут текстовый вывод.

Что происходит, когда вы переключились на `Ctrl+Alt+F1`

1. **Экран приветствия (Процесс `agetty`):**  
    Когда вы только переключились на черный экран, ядро запускает на этом терминале специальную системную службу — `agetty`. Её единственная задача — написать на экране слово `login:` и ждать, пока вы введете имя пользователя. Она еще не умеет выполнять команды вроде `ls` или `mkdir`.
2. **Проверка пароля (Процесс `login`):**  
    Вы ввели логин, `agetty` делает `exec` и превращается в утилиту `login`. Она спрашивает пароль, сверяет его хэш с базой данных в файле `/etc/shadow` и проверяет, пустить ли вас в систему.
3. **Рождение Оболочки:**  
    Утилита `login` заглядывает в паспорт вашего пользователя (файл `/etc/passwd`). Там в самом конце строки написано, какую оболочку вы предпочитаете (например, `/bin/bash` или `/bin/zsh`).  
    Утилита `login` делает финальный `execve("/bin/zsh")`. Процесс перерождается, и перед вами появляется знакомая строка ввода

#### Что такое Оболочка (Shell), Bash и Zsh

Терминал (`TTY`/`PTS`) сам по себе ничего не умеет — он умеет только передавать байты от клавиатуры к ядру и обратно. Ему нужен собеседник. Этот собеседник — **Оболочка (Shell)**.

**Оболочка (Shell)** — это обычная программа (процесс), которая запускается внутри терминала, слушает дескриптор `0` (`stdin`), принимает от вас текстовые команды, парсит их и делает системные вызовы `fork` и `exec` для их запуска.

- **Bash (Bourne Again Shell):** Самая популярная и стандартная оболочка в мире Linux. Она установлена по умолчанию почти во всех дистрибутивах (Ubuntu, Debian, CentOS). Это надежный, проверенный временем стандарт.
- **Zsh (Z Shell):** Более современная оболочка. Она совместима с синтаксисом Bash, но имеет огромный набор улучшений для удобства человека (красивое автодополнение по кнопке Tab, поддержка тем оформления, плагины вроде подсветки синтаксиса на лету). Именно Zsh сегодня является оболочкой по умолчанию в macOS и дистрибутиве Arch Linux.

Чтобы поменять оболочку по умолчанию (например, на Zsh), используйте стандартную команду **`chsh`** (Change Shell)

```bash
chsh -s /bin/zsh
```

Список разрешенных оболочек можно посмотреть командой `cat /etc/shells`

#### Зачем нужно это разделение? (В чем смысл оболочки?)

Если бы терминал умел выполнять команды сам, это намертво связало бы логику ввода-вывода с логикой управления программами. Разделение «Канал связи (Терминал)» + «Интерпретатор (Оболочка)» дает три мощнейших преимущества:

1. **Свобода выбора:**  
    Один пользователь на сервере может любить строгий и классический **Bash**, второй — продвинутый **Zsh** с плагинами, а третий — современный **Fish** с автодополнением. Поскольку терминал — это просто «труба» для байт, каждый из них может запустить внутри этой трубы ту оболочку, которая ему нравится.
2. **Язык программирования:**  
    Оболочка — это не просто запускалка команд, это полноценный интерпретируемый язык программирования. Внутри Bash или Zsh есть циклы `for`, условия `if-else` и функции. Терминалу (экрану и клавиатуре) об этих циклах знать ничего не нужно — их парсит и выполняет процесс оболочки.
3. **Скрипты автоматизации:**  
    Поскольку оболочка — это отдельная программа, вы можете записать команды в текстовый файл (`script.sh`) и попросить Bash выполнить его вообще **без открытия терминала** (например, в фоне по расписанию через Cron)

#### Что такое удаленное подключение по SSH и как оно работает

**SSH (Secure Shell / Безопасная оболочка)** — это сетевой протокол и программа, которая позволяет безопасно подключиться к терминалу удаленного компьютера (сервера) через интернет.

Когда вы пишете в консоли своего ноутбука команду `ssh user@server.com`, запускается красивый танец дескрипторов и сетевых сокетов:

```text
 ВАШ НОУТБУК (Клиент)                   УДАЛЕННЫЙ СЕРВЕР
+--------------------+                 +-----------------------------------+

| Эмулятор терминала |                 | Системный демон sshd (Демон)       |
| (Слушает клавиатуру)                 | (Слушает сеть на 22 порту)        |
+---------+----------+                 +-----------------+-----------------+

          |                                              |
          v                                              v
+---------+----------+      Зашифрованный      +---------+-----------------+

| Процесс ssh-клиента| ═══════ Сетевой ═══════>| Создает виртуальный       |
|                    | <══════  Сокет  ═══════ | терминал /dev/pts/1       |
+--------------------+                         +---------+-----------------+
                                                         |
                                                         v
                                               +---------+-----------------+

                                               | Запускает внутри него     |
                                               | оболочку Bash (процесс)   |
                                               +---------------------------+
```


1. **На стороне сервера** постоянно запущен фоновый процесс-демон **`sshd`** (Secure Shell Daemon). Он открывает сетевой сокет и слушает порт 22, ожидая гостей.
2. **Вы на ноутбуке** вводите текст. Ваш локальный эмулятор терминала передает эти байты процессу `ssh` (клиенту).
3. Процесс `ssh` шифрует эти байты и отправляет их по сети через сокет на сервер.
4. **Демон `sshd` на сервере** принимает зашифрованные байты, расшифровывает их и просит ядро Linux создать новый псевдо-терминал `/dev/pts/1`.
5. После этого `sshd` делает `fork` и `exec` и **запускает внутри этого `/dev/pts/1` удаленную оболочку Bash**.
6. Теперь, когда вы нажимаете клавишу на ноутбуке, буква летит через океан по сети, демон `sshd` ловит её, втыкает в дескриптор `0` удаленного Bash, удаленный Bash выполняет команду, выдает ответ в дескриптор `1`, `sshd` забирает этот ответ, шифрует, шлет вам обратно, и ваш локальный терминал рисует текст на экране

#### Как выглядит работа с командной строкой:

```
$(1) command(2) -x(3) -x args(4) --long-options(5) args(6) ... (enter(7))

1 - Приглашение к вводу команды ($ у обычного пользователя, # у root)
2 - Сама команда
3 - Опции команды стандартные
4 - Опции со своими собственными аргументами
5 - Длинные опции, состоящие в основном из слов
6 - Аргументы, в bash $1, $2... (std::cin)
7 - Управляющий символ:
Ctrl+J - аналог ENTER
Ctrl+H - удаление символа
Ctrl+W - удаление слова
Ctrl+U - удаление строки
Ctrl+C - штатное прерывание
Ctrl+\ - аварийное прерывание
Ctrl+D - завершение ввода
Ctrl+R - поиск в истории командного интерпретатора

[] - если опции обернуть в квадратные скобки, то они будут необязательны
{-x | --long-options} - взаимоисключение

Посмотреть опции команды можно с помощью: <command> --help, -h или man <command> 
```

Настройки, влияющие на поведение команд, можно задавать с помощью переменных окружения: `env`

## Философия Linux

##### Основные принципы Unix-философии:

1. "Всё есть файл" - даже устройства, процессы, сокеты представлены как файлы
2. Одна программа - одна задача - программы должны делать одно, но хорошо
3. Комбинируемость - вывод одной программы может быть входом для другой
4. Текстовые интерфейсы - предпочтение тексту, а не GUI
5. Прозрачность - конфиги в читаемом текстовом формате

##### Почему Linux?
- Open Source - можно посмотреть и изменить любой код
- Стабильность - годами работает без перезагрузок
- Гибкость - можно собрать систему под любые задачи
- Автоматизация - всё управляется через CLI и скрипты
- Контейнеры - Docker и Kubernetes построены на возможностях Linux

## Ядро Linux

**Ядро** состоит из остова - `/boot/vmlinuz`, модулей `/lib/modules/*.ko` и [[init-process|ядерных процессов]]. Оно занимается:
- Управлением процессами (CPU scheduling)
- Управлением памятью (virtual memory, paging)
- Управлением устройствами (драйверы)
- Файловыми системами (VFS - Virtual File System)
- Сетевым стеком (TCP/IP, netfilter)

##### Версии ядра

```
6.1.25-gentoo
│ │ │
│ │ └── patch level (исправления)
│ └── major version (основная версия)
└── kernel version (версия ядра)
```

##### Модули ядра

Динамически загружаемые части ядра:

```bash
lsmod                   # Список загруженных модулей
modprobe module_name    # Загрузить модуль
modinfo module_name     # Информация о модуле
```

## Стандартные потоки ввода/вывода

#### Три стандартных потока

|Поток|FD|Назначение|Перенаправление|
|---|---|---|---|
|stdin|0|Ввод (клавиатура)|`<`|
|stdout|1|Вывод (экран)|`>` или `>>`|
|stderr|2|Ошибки (экран)|`2>`|

#### Перенаправление (Redirection)

```bash
# Основные операторы
> file          # Перезаписать stdout в файл
>> file         # Добавить stdout в конец файла
2> file         # Перезаписать stderr в файл
&> file         # stdout и stderr в файл
< file          # Читать из файла в stdin
2>&1            # stderr в то же место, что и stdout
```
#### Примеры:
```bash
ls > list.txt                       # Сохранить вывод
find / -name "*.log" 2> errors.txt   # Ошибки отдельно
command > output.log 2>&1           # Всё вместе
command < input.txt > output.txt    # Чтение из файла, запись в файл
```

#### Pipes (Конвейеры)

Соединяют stdout одной команды с stdin другой:

```bash
ps aux | grep nginx | awk '{print $2}' | xargs kill -9
#     │         │              │             │
#     └──> поток данных ───────┴─────────────┘
```

#### Here Documents

```bash
cat << EOF > config.txt
user nginx;
worker_processes auto;
EOF
```

## Планировщик задач

#### Cron (периодические задачи)

Формат: `минута час день месяц день_недели команда`

```text
* * * * * command
│ │ │ │ │
│ │ │ │ └── день недели (0-7, 0=воскресенье)
│ │ │ └──── месяц (1-12)
│ │ └────── день месяца (1-31)
│ └──────── час (0-23)
└────────── минута (0-59)

Специальные значения:

@reboot     - при загрузке
@daily      - 0 0 * * *
@hourly     - 0 * * * *
@weekly     - 0 0 * * 0
@monthly    - 0 0 1 * *
```

#### Управление:

```bash
crontab -e              # Редактировать для текущего пользователя
crontab -l              # Посмотреть
crontab -r              # Удалить
```
#### Системный cron:

```
/etc/crontab
/etc/cron.d/
/etc/cron.hourly/
/etc/cron.daily/
/etc/cron.weekly/
/etc/cron.monthly/
```

#### At (одноразовые задачи)

```bash
at now + 5 minutes       # Через 5 минут
at> /path/to/script.sh
at> Ctrl+D
atq                     # Список задач
atrm job_number         # Удалить задачу
```

#### Systemd timers (современная замена cron)

```bash
systemctl list-timers
```
