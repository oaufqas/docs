##### **`Nmap` (Network Mapper)** — швейцарский нож для сетевого аудита. Позволяет узнать, какие устройства в сети, какие порты открыты и какие сервисы там работают.

**Основные флаги:**

- `-sn` — «ping scan». Быстро проверить, какие хосты в сети активны, без сканирования портов.
- `-sS` — сканирование TCP портов
- `-sU` — сканирование UDP портов
- `-p-` — сканирование всех портов

```bash
nmap -sn 192.168.1.0/24
# `-F` — быстрый скан (только 100 самых популярных портов).
nmap -F 192.168.1.1
# `-p` — указать порты для сканирования.
nmap -p 80,443 192.168.1.1       # несколько портов
nmap -p 1-1000 192.168.1.1       # диапазон
# `-sV` — определить версию сервиса, работающего на порту.
nmap -sV 192.168.1.1
# `-O` — попытаться определить операционную систему хоста (часто требует прав root).
sudo nmap -O 192.168.1.1
# `-A` — «агрессивный» режим. Включает определение ОС, версий сервисов и трассировку маршрута.
sudo nmap -A 192.168.1.1
```

##  **Взаимодействие по HTTP (cURL)**

**`cURL`** — незаменимый инструмент для передачи данных с URL. Поддерживает кучу протоколов, но для нас главное — HTTP/HTTPS .

**Основные флаги:**

```bash
curl -fsSl https://get.docker.com | sudo bash

# `-f` (`--fail`): Заставляет curl молчать при ошибках сервера. Если страница не найдена (ошибка 404) или сервер недоступен (500), curl не выведет код ошибки в поток, и скрипт `bash` не начнет исполнять сообщение об ошибке как команды.

# `-s` (`--silent`): Тихий режим. Отключает индикатор прогресса (шкалу загрузки) и сообщения о скорости, чтобы они не засоряли консоль и не передавались в `bash`.

# `-S` (`--show-error`): Используется вместе с `-s`. Если загрузка прервется из-за сетевой ошибки, curl всё же покажет сообщение об ошибке. Без этого флага при ошибке соединения вы просто ничего не увидите.

# `-L` (`--location`): Разрешает curl следовать перенаправлениям (редиректам). Если сервер ответит, что скрипт переехал на другой адрес (код 3xx), curl автоматически перейдет по новой ссылке.

# `-I` (или `--head`) — получить только заголовки HTTP-ответа. Отлично для проверки сервера.

curl -I https://example.com
# `-L` — автоматически следовать за редиректами (HTTP 301/302).
curl -L http://example.com
# `-o` — сохранить ответ в файл.
curl -o saved_page.html https://example.com
# `-O` — сохранить файл, используя его имя из URL.
curl -O https://example.com/file.zip
# `-d` — отправить POST-запрос с данными.
curl -d "param1=value1&param2=value2" -X POST http://example.com/api
# `-H` — добавить свой заголовок.
curl -H "Content-Type: application/json" -H "Authorization: Bearer TOKEN" http://example.com/api
# `-v` — подробный вывод. Покажет все детали запроса и ответа (включая SSL-сертификаты).
curl -v https://example.com
```

## **wget** (World Wide Web + get)

**wget** — утилита для скачивания файлов по протоколам HTTP, HTTPS и FTP

**Основные флаги:**

- **`-O`** — сохранить файл под конкретным именем

```bash
wget -O my.zip url
```

- **`-P`** Скачать в указанную папку: `wget -P /home/downloads url`
- **`-c`** **Докачать** частично загруженный файл (continue)
- **`-b`** Запустить скачивание в **фоновом режиме** (background)
- **`-r`** **Рекурсивное** скачивание (по ссылкам на странице)
- **`-l`** Глубина рекурсии (например, `-l2` — только на 2 уровня вглубь)
- **`-i`** Скачать список ссылок из текстового файла: `wget -i links.txt`
- **`-q`** «Тихий» режим (без вывода в консоль)
- **`--limit-rate`** Ограничить скорость загрузки: `--limit-rate=500k`

```bash
# Просто скачать файл:
wget https://example.com
# Скачать весь сайт целиком (зеркало):  
wget -m https://example.com
# Скачать все картинки определённого формата с сайта:  
wget -r -A.jpg, .png https://example.com
# Продолжить прерванную загрузку большого архива:
wget -c https://example.com
# Скачать файл, игнорируя ошибки SSL-сертификата:  
wget --no-check-certificate https://expired-site.com
```

## **DNS-запросы (dig, nslookup)**

**dig (domain information groper)** — самая навороченная утилита для ручных DNS-запросов. Незаменима для диагностики.

**Основные флаги:**


```bash
# Простой A-запрос (по умолчанию):
dig example.com
# `+short` — получить только самое главное (IP-адрес), без лишней информации.
dig +short example.com
# Запрос конкретной записи: MX (почтовые серверы), NS (DNS-серверы) и т.д:
dig example.com MX
dig example.com NS
# `@dns-server` — отправить запрос конкретному DNS-серверу (вместо системного по умолчанию).
dig @8.8.8.8 example.com
# `-x` — обратный DNS-запрос (по IP узнать домен).
dig -x 8.8.8.8
# `+trace` — проследить весь путь запроса от корневых DNS-серверов до ответа.
dig +trace example.com
```

## **Нагрузочное тестирование (wrk)**

**wrk** — современный и мощный инструмент для генерации нагрузки и измерения производительности веб-серверов. Может использовать Lua-скрипты для сложных сценариев.

**Основные флаги:**

- `-t` — количество потоков (threads).

- `-c` — количество одновременных соединений (connections).

- `-d` — длительность теста.

```bash
# 12 потоков, 400 соединений, тест 30 секунд
wrk -t12 -c400 -d30s https://example.com/
```

- `--latency` — вывести детальную статистику по задержкам (распределение в процентах).

```bash
wrk -t8 -c200 -d60s --latency https://example.com/api
```

- `-s` — путь к Lua-скрипту для кастомизации запросов (POST данные, заголовки, динамические пути).

```bash
# Пример для POST-запроса
wrk -t4 -c100 -d30s -s post.lua https://example.com/api/login
```

##  **Мониторинг портов и соединений (ss / netstat)**

Эти утилиты показывают, что происходит с сетью прямо сейчас. `ss` — новая и более быстрая замена `netstat`.

### **ss (socket statistics) — предпочтительнее**

- `-t` — показать TCP-сокеты.
- `-u` — показать UDP-сокеты.
- `-l` — только слушающие (listening) сокеты.
- `-p` — показать процесс, который использует сокет. `(нужны права root)`
- `-n` — не резолвить имена (показывать IP и порты цифрами).

```bash
ss -tuln       # показать все TCP и UDP порты, которые слушаются
ss -tup        # показать все установленные соединения с процессами
```


### **netstat (network statistics)**

Делает почти то же самое, но работает медленнее на системах с большим количеством соединений.

- `-a` — все сокеты (и слушающие, и с соединениями).
- `-t` / `-u` — TCP/UDP.
- `-l` — только слушающие.
- `-n` — в числовом формате.
- `-p` — показать процесс. `(нужны права root)`

```bash
netstat -tuln   # показать все открытые порты
netstat -anp    # показать все соединения с процессами (аналог ss -tup)
netstat -i      # статистика по сетевым интерфейсам
```

## **Диагностика и конфигурация сети**

### **ip** — современная замена ifconfig, route и arp

Главный инструмент для настройки сетевых интерфейсов, маршрутизации и туннелей. Входит в пакет `iproute2`.

**Синтаксис:** `ip [options] object [command]`

- **Просмотр всех интерфейсов и IP-адресов**:

```bash
ip addr           # или ip a
ip -4 addr        # только IPv4
ip -6 addr        # только IPv6
```

- **Управление состоянием интерфейса**:

```bash
ip link set eth0 up      # включить интерфейс
ip link set eth0 down    # выключить
ip link set eth0 mtu 9000 # изменить MTU
```

- **Добавление/удаление IP-адреса**:

```bash
ip addr add 192.168.1.100/24 dev eth0
ip addr del 192.168.1.100/24 dev eth0
```

- **Просмотр и управление маршрутами**:

```bash
ip route                  # показать таблицу маршрутизации
ip route show default     # показать маршрут по умолчанию
ip route add default via 192.168.1.1 dev eth0  # добавить шлюз
ip route get 8.8.8.8      # показать, через какой интерфейс пойдет трафик к 8.8.8.8 [citation:7]
```


### **ifconfig** — классика (устаревшая, но всё ещё встречается)

Раньше была основной утилитой для настройки сети. Сейчас заменена на `ip`.

- **Показать все интерфейсы**:

```bash
ifconfig
ifconfig -s      # краткая статистика [citation:8]
```

- **Назначить IP**:

```bash
ifconfig eth0 192.168.1.100 netmask 255.255.255.0 up
```


### **ethtool** — диагностика и настройка сетевой карты

Позволяет смотреть и менять параметры физического сетевого адаптера (скорость, дуплекс, автосогласование).

- **Просмотреть текущие параметры**:
```bash
ethtool eth0
```

- **Посмотреть статистику драйвера** (ошибки, коллизии):

```bash
ethtool -S eth0
```

- **Изменить скорость/дуплекс**:

```bash
ethtool -s eth0 speed 1000 duplex full autoneg off
```


### **iw** — работа с Wi-Fi

Аналог `ethtool` и `ip` для беспроводных интерфейсов.

- **Показать информацию о беспроводном устройстве**:

```bash
iw dev
```

- **Просканировать доступные сети**:

```bash
iw wlan0 scan | grep SSID
```

##  **Диагностика сетевого пути**

### **traceroute** / **tracepath** — отследить путь пакетов

Показывает все промежуточные узлы (хопы), через которые проходят пакеты до целевого хоста. Незаменим для поиска места потери пакетов или большой задержки.

- **Базовое использование**:

```bash
traceroute google.com
traceroute 8.8.8.8
```

- **Изменить максимальное количество хопов**:

```bash
traceroute -m 30 google.com
```

- **`tracepath`** — упрощенная версия, не требует root-прав:

```bash
tracepath yandex.ru
```


### **mtr** — комбинация ping + traceroute в реальном времени

Очень мощная утилита. Она постоянно опрашивает каждый хоп на пути и показывает статистику потерь и задержек в реальном времени.

- **Запустить в интерактивном режиме**:

```bash
mtr google.com
```

- **Сгенерировать отчёт** (например, для тикета в поддержку):

```bash
mtr -r -c 100 google.com > mtr-report.txt
```

- **Принудительно использовать TCP или UDP вместо ICMP**:

```bash
mtr --tcp google.com
mtr --udp google.com
```

### **arp** — таблица соответствия IP ↔ MAC-адресов

Протокол ARP используется для преобразования IP-адресов в физические MAC-адреса в локальной сети.

- **Показать текущую ARP-таблицу**:

```bash
arp -a
arp -n        # без разрешения имён (только IP)
```

- **Удалить запись**:

```bash
arp -d 192.168.1.100
```

### **nc (netcat)** — «швейцарский нож» TCP/UDP

Позволяет создавать произвольные TCP/UDP соединения, слушать порты, передавать файлы.

- **Проверить, открыт ли порт на удалённом хосте**:

```bash
nc -zv example.com 80
```

`-z` — просто сканировать, не отправляя данные; `-v` — подробно.

- **Слушать порт (создать простой сервер)**:

```bash
nc -l 8080
```

- **Отправить файл**:

```bash
nc -l 9999 > received.zip   # на приёмнике
nc receiver_ip 9999 < file.zip   # на отправителе
```

### **tcpdump** — консольный сниффер пакетов

Позволяет захватывать и анализировать сетевой трафик прямо в консоли.

- **Посмотреть список интерфейсов**:

```bash
tcpdump -D
```

- **Захватить 10 пакетов на интерфейсе eth0**:

```bash
tcpdump -i eth0 -c 10
```

- **Фильтровать по хосту**:

```bash
tcpdump host 8.8.8.8
tcpdump src host 192.168.1.100   # только от этого хоста
tcpdump dst host google.com      # только к этому хосту
```

- **Фильтровать по порту и протоколу**:

```bash
tcpdump port 80
tcpdump tcp port 443
```

- **Сохранить захват в файл для анализа в Wireshark**:

```bash
tcpdump -i eth0 -w capture.pcap
```

## **Нагрузочное тестирование и производительность**

### **iperf / iperf3** — замер максимальной пропускной способности

Позволяет измерить реальную скорость сети между двумя машинами.

- **На сервере (приёмник)**:

```bash
iperf3 -s
```

- **На клиенте (отправитель)**:

```bash
iperf3 -c server_ip
```

- **Изменить длительность теста**:

```bash
iperf3 -c server_ip -t 60
```

### Сеть в Linux

##### Сетевые интерфейсы

Именование:

- `eth0`, `eth1` - Ethernet (старое)
- `enp0s3`, `ens33` - предсказуемые имена
- `lo` - loopback (localhost)

```bash
ip link                    # Список интерфейсов
ip addr show               # IP адреса
ip route show              # Таблица маршрутизации
```

#### Netfilter и iptables

Таблицы:

- **filter** - фильтрация пакетов
- **nat** - трансляция адресов
- **mangle** - изменение пакетов
- **raw** - особые настройки

Цепочки:

- **INPUT** - входящие на сервер
- **OUTPUT** - исходящие с сервера
- **FORWARD** - транзитные
- **PREROUTING** - до маршрутизации
- **POSTROUTING** - после маршрутизации

#### Socket (сокеты)

Точка соединения между процессами:
- **Unix domain sockets** - для IPC внутри системы
- **TCP/UDP sockets** - для сети


```bash
ss -tulpn                 # Просмотр открытых портов
netstat -tulpn            # То же, старой версией
lsof -i :80               # Кто на порту 80
```
