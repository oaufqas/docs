### Переменные окружений

Переменные окружения есть в каждой популярной ОС (win, linux, mac etc.). Они существуют для адаптации ПО, удобства настройки и автоматизации, **это пары ключ=значение**. Например **переменная $PATH в линукс или %PATH% в виндовс** хранит путь к папкам, где система ищет программы при вводе команды. **Скрытие конфиденциальных данных**: Хранение ключей API или паролей к базам данных (часто используется в контейнеризации).

Окружение - что то, что окружает наше запущенное приложение, например операционная система. В этом окружении могут задаваться какие либо переменные, которые могут использоваться в нашем приложении. Есть уровни переменных окружения:
1. Системные - Уровень операционной системы - доступ к ним есть везде, у всех пользователей.
2. Пользовательские - Уровень текущего пользователя, у каждого могут быть свои заданные переменные.
3. Сеансовые - Уровень текущего сеанса, существуют только внутри конкретного окна терминала или сессии и исчезают после их закрытия.
4. Программные - Уровень запуска приложения, она может быть запущенна со своими переменными

## Где хранятся переменные окружения в Linux
- Временные: export VAR=value (живут в текущей сессии).
- Постоянные (для пользователя): ~/.bashrc, ~/.bash_profile, ~/.profile.
- Постоянные (системные): /etc/environment, /etc/profile, скрипты в /etc/profile.d/.
- Просмотр: Команды printenv или env. 

## Где хранятся переменные окружения в Windows
- Системные и Пользовательские: Хранятся в системном реестре (Windows Registry).
- Графический интерфейс: Свойства системы -> Дополнительно -> Переменные среды.
- Командная строка: set (для текущей сессии), setx (для постоянной записи).
- Синтаксис использования: Переменные заключаются в знаки процента, например: %PATH% или %USERPROFILE%

Например **setx SSLKEYLOGFILE "C:\temp\sslkeys.log"** - переменная, которая сохраняет секреты TLS-шифрований в браузерах.
