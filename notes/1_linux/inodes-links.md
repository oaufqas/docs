## inode (Index Node) - индексный дескриптор

Файловая система опирается на метаданные - данные о данных. В древовидной структуре каталогов хранятся лишь **названия файлов** и **номера inode**

```
newfolder --> filename.txt
              235527628429 (inode)
```

Каждый файл имеет inode - структуру с метаданными:

- Права доступа
- Владелец и группа
- Размер
- Временные метки (atime, ctime, mtime)
- Количество жестких ссылок
- Указатели на блоки данных на жестком диске

Для inode есть своя таблица, занимающая место на диске, и они могут закончится, тогда создавать новые файлы не получится, даже если есть место на диске

Если файл слишком маленький, то его данные могут хранится в таблице inode. Например все симлинки хранятся именно в этой таблице

#### Освобождение номеров inode

Как только счетчик жестких ссылок на этот Inode в системе падает до нуля (то есть удалены все имена, связанные с ним), Linux делает следующее:

1. Помечает этот номер Inode в специальной битовой карте как **«свободный»**. Теперь система может выдать этот паспорт новому файлу.
2. Помечает блоки данных на диске, где лежал контент, как **«свободные для записи»**.  
    _(Сами данные физически не затираются нулями сразу ради скорости — именно поэтому случайно удаленный файл иногда можно восстановить, если на диск еще ничего не успели записать поверх)._


`ls -i file`              # Показать inode файла
`stat file`               # Детальная информация о файле
`mkfs.ext4`               # Управление inode
`df -hi`                     # Информация об inode на ФС, сколько всего inode в системе.

`find / -xdev -type d -print0 | xargs -0 -I {} sh -c 'echo $(find "{}" -maxdepth 1 | wc -l) "{}"' | sort -n -r | head -n 20`    # Команда для поиска папок, с наибольшим колличеством занятых inode.

### Ссылки — это указатели на файлы или папки.

 Вместо того чтобы копировать огромный файл и занимать место, вы можете создать ссылку на него.
##### Жесткая ссылка (Hard Links)

Жесткая ссылка — это **еще одно имя для того же самого Inode**. 

- **Как это устроено:** Создавая жесткую ссылку, вы просто добавляете новую запись в таблицу директории (Новое Имя <—> Старый Номер Inode).
- **Размер:** Занимает 0 байт на диске, так как блоки данных не дублируются.
- **Ограничения:**
    1. Нельзя создать ссылку на **папку** (чтобы избежать бесконечных циклов в системе).
    2. Нельзя создать ссылку на файл, который лежит на **другом диске или другой файловой системе** (ведь на другом диске таблица Inode своя, и там этот номер может принадлежать чужому файлу).
- **Удаление:** Если вы удалите оригинал, файл **не исчезнет**. Он будет доступен по жесткой ссылке. Данные удалятся только тогда, когда вы сотрете _все_ жесткие ссылки на этот Inode (счетчик ссылок в паспорте файла станет равен 0).
- Любое имя файла в любой папке - это и есть hardlink, системе без разницы что было первое. У одной inode могут быть разные именя жестких ссылок.
- При изменении прав на одну жесткую ссылку, они изменятся и на другую, потому что права хранятся в inode, а она у жестких ссылок одна.
- При создании папки, у нее есть сразу 2 hardlink'а:
	1. **Первая ссылка:** Имя `my_folder` в родительской директории.
	2. **Вторая ссылка:** Скрытое имя `.` (точка) внутри самой `my_folder`.
	3. Каждая **поддериктория** в этой папке будет дополнительной ссылкой

`ln target link_name`
`ls -li`                  # Одинаковый inode


##### Символические ссылки (Symlinks / Мягкие ссылки)

Символическая ссылка — это **отдельный маленький файл особого типа**, внутри которого написан текстовый путь к оригинальному файлу. Это полный аналог «Ярлыков» из Windows.

- **Как это устроено:** У симлинка создается **свой собственный, новый номер Inode** и свои блоки данных, где записан текст (например: `/var/www/index.html`).
- **Преимущества:**
    1. Можно ссылаться на **любые папки**.
    2. Можно ссылаться на файлы на **других дисках, флешках или даже сетевых папках**.
- **Удаление:** Если вы удалите оригинал, симлинк останется, но он «сломается» (станет подсвечиваться красным в консоли). Ссылка будет вести в никуда. Если удалить сам симлинк — с оригиналом ничего не случится.
- **Fast Symlinks (Быстрые ссылки):** Если путь к оригиналу короткий (меньше 60 символов в ext4), Linux не тратит блоки данных на диске. Он записывает текстовый путь прямо внутрь самого Inode — туда, где обычно хранятся адреса блоков данных. Такая ссылка занимает **0 блоков** на диске и считывается мгновенно.
- **Slow Symlinks (Медленные ссылки):** Если путь длинный (например, больше 60 символов), выделяется стандартный блок данных на диске (обычно 4 КБ), и этот текстовый путь записывается туда.
- Права доступа всегда `lrwxrwxrwx`, проверка идет из прав на сам ссылаемый объект.
- Многое зависит от того, абсолютный путь или относительный.

`ln -s target symlink_name`
`ls -l`                   # Видно куда указывает

Различия:

Жесткая - нельзя на директории, нельзя между ФС
Символическая - можно на всё, если удалить оригинал - ссылка сломается
