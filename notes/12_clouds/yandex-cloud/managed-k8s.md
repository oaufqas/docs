### Создание kubernetes cluster

#### 1. Подготовка (Сеть и Права)

- **Создание сети:** Перейди в раздел **Virtual Private Cloud** -> **Создать сеть**. Любое название, например`k8s-network`. Убедись, что галочка «Создать подсети» стоит. Публичные IP-адреса -> Зарезервировать публичный IP-адрес
- **Создание сервисного аккауна:**  **[[iam|Перейди в IAM]]** -> **Создать сервисный аккаунт**.
    - Имя: `k8s-sa`.
    - Роли: Добавь две роли — `k8s.clusters.agent` и `vpc.publicAdmin` (это минимум для старта), но для обучения проще дать `editor` на весь каталог.

#### 2. Создание Кластера (Master)

1. **Managed Service for Kubernetes**.
2. **Создать кластер**.
3. **Настройки:**
    - **Зональный**: выбери одну зону (например, `ru-central1-a`).
    - **Версия**: по умолчанию.
    - **Облачная сеть**: выбери свою `k8s-network`.
    - **Сервисные аккаунты**: в обоих полях выбери созданный `k8s-sa`.
    - **Сетевые политики (Network Policies):** Для обучения **выключи**. Если включишь (Calico), Яндекс добавит скрытые ресурсы для их работы. Тебе они понадобятся позже, когда будешь учить безопасность.
	- **Туннельный режим:** Оставь **выключенным**. Это нужно для очень больших и специфичных сетей.
	- **CIDR кластера и сервисов:** Оставь **по умолчанию** (обычно `10.112.0.0/16` и `10.96.0.0/16`). Это внутренние диапазоны IP-адресов для подов и сервисов. Они не пересекаются с твоей домашней сетью.
	- **Маска подсети узлов:** Оставь `/24`. Это значит, что на одну ноду можно "засунуть" до 254 подов — для тебя это более чем достаточно.
4. **Важно:** Включи флаг **Публичный IP-адрес**, чтобы ты мог управлять кластером со своего ПК.
5. Нажми **Создать** и жди 5–10 минут (статус должен стать `Running`).

#### 3. Создание Группы узлов (Worker Nodes)

1. Зайди в свой созданный кластер -> вкладка **Группы узлов** -> **Создать группу узлов**.
2. **Настройки ресурсов:**
    - **Платформа**: Intel Ice Lake.
    - **vCPU**: 2.
    - **RAM**: 4 ГБ.
    - **Прерываемая (Preemptible)**: **ОБЯЗАТЕЛЬНО включи** (экономия 70%). Работает только на конфигурациях - **Intel Ice Lake** (standard-v3), **Intel Cascade Lake** (standard-v2)
3. **Масштабирование**: выбери «Фиксированное» и количество узлов — `1`.
4. **Сетевые настройки**: включи «Публичный IP», чтобы ноды могли ходить в интернет за образами.

#### 4. [[cli|Установка Yandex cli (yc)]] и подключение кластера к kubectl

```bash
# Подключить кластер к локальному kubectl
yc managed-kubernetes cluster get-credentials --id <ID_КЛАСТЕРА> --external
```

#### 5. Установка ingress controller

Самая простая установка ingress - через helm: 

```bash
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx && \ 
helm repo update && \ 
helm install ingress-nginx ingress-nginx/ingress-nginx
# --set controller.service.loadBalancerIP=<ip_addr> Прокинуть в ингресс уже существующий ip адресс
```
