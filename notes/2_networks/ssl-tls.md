## Протоколы TLS/SSL

**SSL (Secure Sockets Layer)** и его преемник **TLS (Transport Layer Security)** — это криптографические протоколы, обеспечивающие безопасную передачу данных поверх транспортного уровня (обычно TCP). Они используются для защиты HTTP (HTTPS), электронной почты (SMTPS, IMAPS), VPN и других протоколов.

#### Основные задачи TLS/SSL:

1. **Аутентификация** – подтверждение подлинности сервера (и опционально клиента) с помощью цифровых сертификатов.

2. **Конфиденциальность** – шифрование данных, чтобы их нельзя было прочитать при перехвате.

3. **Целостность** – защита от модификации данных в пути (с помощью кодов аутентификации сообщений, MAC).


#### Как работает TLS (упрощённо):

1. **Рукопожатие (Handshake):**
    
    - Клиент и сервер согласовывают версию протокола, набор шифров (cipher suite).
    - Сервер отправляет свой цифровой сертификат (содержит публичный ключ).
    - Клиент проверяет сертификат (с помощью доверенных центров сертификации, CA).
    - Генерируется сеансовый ключ (обычно с использованием алгоритмов Диффи-Хеллмана или RSA) для симметричного шифрования.
        
2. **Переключение на защищённое соединение:** все дальнейшие данные шифруются сеансовым ключом.
    
3. **Завершение сеанса:** корректное закрытие соединения с предупреждением.

#### Основные версии:

- **SSL 2.0, 3.0** – устаревшие, небезопасные.
- **TLS 1.0, 1.1** – также устарели, не рекомендуются.
- **TLS 1.2** – широко используется, надёжен.
- **TLS 1.3** – современная версия, улучшенная производительность и безопасность (сокращённый handshake, удаление устаревших алгоритмов).

#### Компоненты:

- **Сертификаты X.509** – электронные документы, удостоверяющие принадлежность публичного ключа владельцу. Выдаются центрами сертификации (CA: Let's Encrypt, DigiCert, GlobalSign).

- **Цепочка доверия** – сертификат сервера подписан промежуточным CA, который подписан корневым CA (встроенным в доверенные корневые центры ОС или браузера).

- **Асимметричное шифрование** – используется в рукопожатии для безопасного обмена ключами.

- **Симметричное шифрование** – используется для данных сеанса (быстрее).

- **Хэш-функции** – для целостности (HMAC).

### Где используется TLS/SSL:

- HTTPS (HTTP over TLS) – защищённый веб.
- FTPS – защищённый FTP.
- SMTPS, IMAPS, POP3S – защищённая почта.
- VPN (OpenVPN, WireGuard используют другие протоколы, но TLS тоже может применяться).
- Защита API (REST, GraphQL через HTTPS).
