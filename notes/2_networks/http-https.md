**HTTP** (HyperText Transfer Protocol) — это протокол прикладного уровня для передачи данных. Изначально созданный для HTML-документов, сегодня он переносит практически любой контент (текст, фото, видео). 

**HTTPS** (HyperText Transfer Protocol Secure) — это «обертка» безопасности над стандартным HTTP. Он использует шифрование, чтобы никто не смог перехватить или подменить ваши данные (пароли, номера карт). Использует **[[ssl-tls#Как работает TLS (упрощённо)|TLS]]** для шифрования данных пакетов. 

# Ключевые различия
| Характеристика               |       HTTP       |     HTTPS
|-----------------------------|---------------|-----------------------------------|
| Безопасность   |   Данные передаются открытым текстом       |    Данные зашифрованы
| Порт        |    По умолчанию 80      |   По умолчанию 443         |
| Шифрование |    Отсутствует |      Использует TLS (ранее SSL)  |
| Доверие	 |    Браузер помечает сайт как «незащищенный»    |  Замок в адресной строке

### Как работает HTTPS (TLS Handshake)

Чтобы начать защищенный обмен, браузер и сервер проходят процедуру «рукопожатия»:
1. Приветствие: Браузер запрашивает защищенное соединение.
2. Проверка сертификата: Сервер отправляет свой SSL-сертификат. Браузер проверяет через центр сертификации (CA), настоящий ли это сайт.
3. Обмен ключами: Браузер и сервер договариваются о секретном ключе, используя асимметричное шифрование.
4. Защищенная связь: Все дальнейшие данные шифруются этим ключом с помощью быстрого симметричного шифрования. 


## Основные виды (методы) запросов
Методы определяют, что именно нужно сделать с ресурсом: 

- `GET` — запрос данных. Самый частый метод (например, загрузка страницы). Параметры передаются прямо в URL. Безопасный: не должен изменять данные на сервере.
- `POST` — отправка данных на сервер (регистрация, комментарий). Данные передаются в теле запроса. Может менять состояние сервера.
- `PUT` — полное обновление ресурса. Если ресурса нет, он будет создан.
- `PATCH` — частичное изменение ресурса (например, сменить только фамилию в профиле).
- `DELETE` — удаление ресурса.
- `HEAD` — аналогичен GET, но сервер присылает только заголовки без тела. Полезно для проверки веса файла или даты его изменения.
- `OPTIONS` — запрос списка методов, которые поддерживает сервер для данного адреса. 

### Коды статуса ответа:

- **1xx** – информационные
- **2xx** – успех (200 OK, 201 Created)
- **3xx** – перенаправление (301 Moved Permanently, 302 Found, 304 Not Modified)
- **4xx** – ошибка клиента (400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found)
- **5xx** – ошибка сервера (500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable)

## HTTP-заголовки (Headers)

Это метаданные, которые передают дополнительные инструкции или информацию о клиенте и сервере. Они пишутся в формате Имя: Значение. 

### Заголовки запроса (от клиента):

Host: адрес сервера (обязателен в HTTP/1.1).
User-Agent: информация о браузере и ОС пользователя.
Accept: типы контента, которые понимает клиент (например, text/html, application/json).
Authorization: данные для входа (логин/пароль или токен).
Cookie: отправка сохраненных сервером данных обратно (для авторизации). 

##### Пример:

```http
GET /index.html HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

### Заголовки ответа (от сервера):

Content-Type: формат передаваемых данных (например, image/png или text/html).
Content-Length: размер тела ответа в байтах.
Server: информация о ПО сервера (например, nginx).
Set-Cookie: команда браузеру сохранить куку.
Cache-Control: инструкции, как долго браузеру можно хранить этот файл в кэше.

##### Пример:

```http
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...
```
