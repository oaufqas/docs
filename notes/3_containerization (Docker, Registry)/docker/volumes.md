**Тома (volumes)** - Сохранение данных с контейнера на физическом диске.
-v флаг для указания томов. Слева от двоеточия директория/имя volume в файлах докера на хостовой машине, справа, директория находящаяся внутри контейнера.


## Основные типы томов:

- Host Volumes (привязка директории на сервере к директории внутри контейнера)

docker run **-v /opt/mysql_data:/var/lib/mysql** mysql:latest
                    |                    |
      физическая директория     директория внутри контейнера



- Anonymous Volumes (Сохраняется в файлах докера с захэшированным названием) /var/lib/docker/volumes/HASH/_data
 
docker run **-v /var/lib/mysql** mysql:latest
                    |
        директория внутри контейнера



- Named Volumes (Сохраняется в файлах докера с заданным названием) /var/lib/docker/volumes/[NAME]/_data

docker run **-v mysql_data:/var/lib/mysql** mysql:latest
                    |             |
        именованный том       директория внутри контейнера





# Посмотреть содержимое volume если подключенного контейнера нет, можно путем создания нового временногоконтейнера и монтирования в него этого тома:

docker run --rm -it **-v <имя/hash тома>:/data** alpine: latest /bin/bash

В контейнер можно монтировать сразу много томов, одним это не ограничивается.
