**Докер контейнер (Container)**: Запущенный экземпляр образа (процесс). Это изолированная среда, которая потребляет минимум ресурсов, так как использует ядро основной операционной системы.
Докер добавляет поверх неизменяемого образа "тонкий" слой для записи, что позволяет приложению работать и сохранять данные во время сессии (работы контейнера).

Запуск и установка контейнеров происходит командой **docker run [FLAGS] IMAGE [COMMAND]**, не путать с (docker START, который запускает уже существующий, но остановленный контейнер). Запускать можно образы, которые есть локально и с docker registry, если нужного образа локально нет, докер установит его с dockerhub. 
**Основные флаги при запуске:**

-d (Detached): Запускает контейнер в фоновом режиме. Вы вводите команду, и терминал остается свободным.
-it: Связка двух флагов (-i и -t). Позволяет взаимодействовать с контейнером (войти внутрь консоли)
--name: Позволяет задать контейнеру понятное имя (например, --name my-web-server). Если не указать, Docker придумает случайное имя вроде flamboyant_hopper
--rm: Автоматически удаляет контейнер сразу после его остановки. Очень полезно для тестов, чтобы не копить «мусор».

-p [host_port]:[container_port]: Проброс портов.
--network [network_name]: Подключает контейнер к определенной сети (например, чтобы база данных и сайт видели друг друга).

-v [host_path]:[container_path]: Монтирует папку с вашего диска в контейнер
-e (Env): Установка переменных окружения. Пример: -e MYSQL_ROOT_PASSWORD=12345.

-m или --memory: Лимит оперативной памяти (например, --memory="512m").
--cpus: Сколько ядер процессора может использовать контейнер (например, --cpus="1.5").


## Команды для управления контейнерами:

docker **ps** - Список запущенных контейнеров (добавьте -a, чтобы увидеть и остановленные).
docker **stop** my-app - Мягкая остановка контейнера (сигнал SIGTERM).
docker **start** my-app - Запуск ранее остановленного контейнера.
docker **restart** my-app - Перезагрузка контейнера
docker **kill** my-app - Принудительная остановка
docker **exec** -it my-app [bash] - Выполнение команды внутри запущенного контейнера.
docker **logs -f** my-app - Просмотр вывода (логов) приложения (-f для слежения в реальном времени).
docker **inspect** my-app - Получение всей детальной информации (IP, конфиги, пути).
docker **rm** my-app - Удаление остановленного контейнера (добавьте -f для принудительного удаления запущенного)
docker **container prune** - Быстрая очистка всех контейнеров
docker **top** my-container - Показать процессы контейнера
