**[[notes/9_infrastructure/terraform/installation|Terraform]]** — это open-source инструмент для автоматизированного управления ИТ-инфраструктурой. Если говорить проще, он позволяет создавать и изменять серверы, базы данных и сети с помощью текстовых файлов, а не через ручную настройку в консоли провайдера. 

В основе Terraform лежит концепция **Infrastructure as Code (IaC)** — инфраструктура как код: 
- **Декларативный подход**: Вы описываете в конфигурационном файле _конечный результат_ (например, «мне нужно 3 сервера и база данных»), а Terraform сам вычисляет, какие действия нужно предпринять, чтобы это реализовать.
- **[[hcl-syntax|Язык HCL]]**: Конфигурации пишутся на языке _HashiCorp Configuration Language_, который легко читать и человеку, и машине.
- **Идемпотентность**: Инструмент проверяет текущее состояние ресурсов и вносит только те изменения, которые необходимы для достижения целевого состояния. 

#### Основные возможности

1. **Мультиоблачность**: Работает с сотнями провайдеров (AWS, Azure, Google Cloud, Yandex Cloud, VK Cloud, Proxmox, GitHub и др.) через специальные плагины — [[providers|провайдеры]].
2. **[[state|Управление состоянием (State)]]**: Terraform запоминает текущую конфигурацию инфраструктуры в специальном файле `terraform.tfstate`, что позволяет отслеживать изменения и безопасно их обновлять.
3. **Безопасность изменений**: С помощью команды `terraform plan` можно заранее увидеть список всех изменений (что будет создано, изменено или удалено), прежде чем применять их на практике. 
4. **[[workspaces|Workspaces]]:** Использование одного и того же кода в разных средах.

Часто Terraform используют вместе с инструментами вроде **[[notes/7_configuration/ansible/basics|Ansible]]**: Terraform создает «железо» (виртуальные машины, сети), а Ansible настраивает внутри них ПО и операционную систему. В отличие от Ansible, Terraform сохраняет состояние (в файле Tfstate).

#### Файловая структура проекта

В минимальном виде проект Terraform состоит из текстовых файлов с расширением `.tf`. По общепринятому стандарту их делят на:

- `main.tf`: основной файл, где описываются ресурсы (серверы, сети).
- `variables.tf`: здесь объявляются переменные для гибкости кода.
- `outputs.tf`: описание данных, которые Terraform выведет после завершения работы (например, IP-адрес созданного сервера).
- `providers.tf`: настройки подключения к провайдерам (токены, регионы).
- `terraform.tfstate`: файл состояния, в котором Terraform «запоминает», что он уже создал. **Его нельзя редактировать вручную**. 

#### Основные сущности (Логика)

- **[[providers|Провайдеры (Providers)]]**: плагины для взаимодействия с API облаков (AWS, Yandex Cloud, Azure и др.).
- **[[hcl-syntax|Ресурсы (Resources)]]**: конкретные объекты инфраструктуры, которыми мы управляем (виртуальная машина, диск, DNS-запись).
- **[[notes/9_infrastructure/terraform/modules|Модули (Modules)]]**: контейнеры для группы ресурсов, которые позволяют повторно использовать один и тот же код в разных проектах.

### Структура проекта terraform:

- `main.tf` - Описание провайдеров, ресурсов, datasource
- `variables.tf` - Хранение обычных переменных
- `outputs.tf` - Определение переменных на output
- `\*.tfvars` - Можно указать значение переменных из variables (самостоятельно не подхватывает, нужно указать при `terraform apply`) Для того, чтобы использовать переменные с файлов `\*.tfvars`, необходимо их объявить в `variables.tf`

Терраформ съедает все файлы из директории вызова.
