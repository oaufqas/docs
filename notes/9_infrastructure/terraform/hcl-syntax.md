Язык **HCL** (HashiCorp Configuration Language) специально разработан так, чтобы быть одновременно понятным для человека и удобным для чтения машиной. Он похож на JSON, но более гибкий и «чистый».

### Вот из чего состоит синтаксис HCL:

##### Блоки (Blocks) - Это основной структурный элемент. Блок имеет тип, метки и тело в фигурных скобках.

```hcl
// тип_блока "метка1" "метка2"
resource "aws_instance" "web_server" {
  # Внутри блока находятся аргументы
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}
```

##### Аргументы (Arguments) - Это пары `ключ = значение`. Именно через них вы настраиваете ресурсы.

- **Важно:** В HCL используется знак `=`, а не двоеточие.

##### Типы данных - HCL поддерживает стандартные типы:

- **Строки:** `"hello"` (всегда в двойных кавычках).
- **Числа:** `42`, `3.14`.
- **Булевы значения:** `true`, `false`.
- **Списки (Lists):** `["us-east-1a", "us-east-1b"]`.
- **Карты (Maps/Objects):** `{ name = "admin", port = 80 }`.

##### Выражения и Интерполяция - Вы можете подставлять значения одних ресурсов в другие. Это делает конфиг динамическим.

```hcl
resource "aws_instance" "example" {
  # Обращение к переменной
  instance_type = var.instance_size 
  
  # Интерполяция (вставка в строку)
  tags = {
    Name = "Server-${var.environment}"
  }
}
```

##### Условные выражения (Ternary) - Работают по принципу `условие ? значение_если_да : значение_если_нет`.

```hcl
count = var.create_server ? 1 : 0
```

##### Функции - Terraform встроено много функций (для работы со строками, списками, датами), но вы **не можете** писать свои.

- Пример: `tags = lower(var.user_name)` (приведет имя к нижнему регистру).

##### Циклы - HCL позволяет создавать ресурсы массово:

- **Count:** простой счетчик (`count = 3` создаст 3 одинаковых ресурса).
- **For_each:** итерация по списку или карте (более продвинутый способ, позволяет обращаться к каждому элементу через `each.key` и `each.value`).

### Синтаксис Terraform

Синтаксис Terraform — это структура конкретных блоков, из которых строится инфраструктура. Основное:

- Resources (Ресурсы)

Это основной строительный блок. Вы используете их, когда хотите, чтобы Terraform **создал что-то новое** или изменил существующее.

- **Действие:** Создать, Обновить, Удалить.
- **Пример:** `resource "yandex_vpc_network" "my_net" {}` — «Яндекс, создай мне новую сеть».

- Data Sources (Источники данных)

Они используются, когда вам нужно **получить информацию** о чем-то, что уже существует в облаке (и, возможно, создано не через Terraform, а руками в консоли или другим отделом).

- **Действие:** Найти, Прочитать.
- **Пример:** `data "yandex_vpc_network" "existing_net" { name = "default" }` — «Яндекс, найди сеть с именем `default` и скажи мне её ID, чтобы я мог её использовать».

```hcl
# 1. Сначала "читаем" данные о существующей сети по её имени
data "yandex_vpc_network" "prod_net" {
  name = "production-network" # Это имя сети, которая уже есть в облаке
}

# 2. Используем полученный ID в ресурсе кластера
resource "yandex_kubernetes_cluster" "my_cluster" {
  network_id = data.yandex_vpc_network.prod_net.id # Берем ID из data source
  # ... остальные настройки
}
```

1. Блок `terraform` (Настройки проекта) - здесь указываются требования к версии самого Terraform и список необходимых провайдеров.

```hcl
terraform {
  required_version = ">= 1.5.0" # Минимальная версия терраформа

  required_providers {
    yandex = {
      source  = "yandex-cloud/yandex"
      version = "~> 0.90" # Фиксация версии провайдера
    }
  }

  backend "s3" { # Настройка хранения State (удаленно)
    bucket = "my-tf-state"
    key    = "prod/terraform.tfstate"
  }
}
```

2. Блок `provider` (Подключение) - настройка конкретного провайдера (авторизация, регион).

```hcl
provider "yandex" {
  token     = var.cloud_token
  cloud_id  = "b1g..."
  folder_id = "b1g..."
  zone      = "ru-central1-a"
}
```

3. Блок `resource` (Главный блок) - описывает объект, который будет создан. У него всегда **две метки**: тип ресурса и ваше локальное имя.

```hcl
#        Тип ресурса (от провайдера)  Ваше имя (внутри кода)
resource "yandex_compute_instance" "web_server" {
  name = "apache-server"

  resources {
    cores  = 2
    memory = 2
  }
  
  # Вложенные блоки (настройки внутри ресурса)
  boot_disk {
    initialize_params {
      image_id = "fd8..."
    }
  }
}
```

4. Блок `data` (Источник данных) - позволяет получить информацию об объектах, которые **уже существуют** в облаке (созданы не вами или в другом проекте).

```hcl
data "yandex_compute_image" "ubuntu" {
  family = "ubuntu-2204-lts"
}

# Теперь мы можем использовать ID этого образа в ресурсе:
# image_id = data.yandex_compute_image.ubuntu.id
```

5. Блок `variable` (Входные данные) - делает ваш код гибким.

```hcl
variable "instance_count" {
  description = "Количество серверов"
  type        = number
  default     = 1
}
```

6. Блок `output` (Выходные данные) - показывает важную информацию после завершения работы (IP-адреса, ID).

```hcl
output "internal_ip" {
  value = yandex_compute_instance.web_server.network_interface.0.ip_address
}
```


7. Блок `locals` (Локальные переменные) - используются как «черновик» или константы внутри модуля, чтобы не повторять сложные вычисления.

```hcl
locals {
  project_name = "MegaApp"
  common_tags = {
    Owner   = "DevOps"
    Project = local.project_name
  }
}
```
