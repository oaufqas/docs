#### 1. Remote Backends (Удаленное хранение State)

По умолчанию Terraform хранит файл состояния на вашем диске. Это плохо, если вы работаете в команде: изменения коллеги затрут ваши.

- **Решение:** Использование **Backend** (например, S3 bucket в AWS или Object Storage в Yandex Cloud).
- **State Locking:** Удаленные бэкенды поддерживают блокировку. Если один человек делает `apply`, второй не сможет начать изменения, пока первый не закончит. Это защищает от порчи инфраструктуры.

#### 2. Зависимости (Depends_on)

Terraform сам строит дерево зависимостей (сначала создает сеть, потом сервер внутри неё). Но иногда зависимость не очевидна программно.

- Используется аргумент `depends_on = [resource_name]`, чтобы явно сказать: «Не создавай этот ресурс, пока не готов вон тот».

#### 3. Жизненный цикл (Lifecycle)

Внутри ресурса можно прописать блок `lifecycle`, который меняет поведение Terraform:

- `prevent_destroy`: запрещает удалять важный ресурс (например, базу с данными), даже если вы запустили `destroy`.
- `create_before_destroy`: сначала создает новую копию ресурса, и только потом удаляет старую (полезно для Zero Downtime).

#### 4. Import (Импорт существующей инфраструктуры)

Если у вас уже есть серверы, созданные вручную «мышкой», вы можете «затащить» их под управление Terraform с помощью команды `terraform import`. Вы описываете ресурс в коде, а затем связываете его с ID реального объекта в облаке.

#### 5. Provisioners (Провижинеры)

Иногда нужно что-то сделать _внутри_ сервера сразу после его создания (например, запустить скрипт).

- `local-exec`: запускает скрипт на вашем компьютере.
- `remote-exec`: запускает скрипт на созданном сервере по SSH.
- _Важно:_ HashiCorp рекомендует использовать их только в крайнем случае, отдавая предпочтение **Cloud-init** или **Ansible**.

#### 6. Sensitive Data (Секреты)

В коде часто нужны пароли или ключи.

- Используйте флаг `sensitive = true` для переменных, чтобы их значения не светились в логах консоли.
- Никогда не храните пароли в `.tf` файлах в открытом виде — используйте переменные окружения или менеджеры секретов (HashiCorp Vault).
