
##### Для чего нужен **Load Balancing** - Распределять нагрузку между несколькими бэкенд-серверами.

- **Upstream блок:**
```nginx
upstream backend {
	server 10.0.0.1:3000 weight=3;
	server 10.0.0.2:3000;
	server 10.0.0.3:3000 backup;
}
server {
	location / {
		proxy_pass http://backend;
	}
}
```

#### Методы балансировки:

- **Round Robin (по умолчанию):** По кругу.
- **Weighted Round Robin** если нужно «подсвечивать» более мощные или более быстрые.
- **Least Connections:** На тот сервер, где меньше всего активных соединений.
- **IP Hash:** Запросы с одного IP всегда на один сервер (для сессий).
- **Weight (вес):** Более мощным серверам можно дать больший вес.
