Observability (наблюдаемость) - это способность системы преластавлять внутреннюю информацию, достаточную для понимания ее состояния и причинно-следственных связей

Основной современный Observability стек-акроним это MELT:

**M - Metrics**. Числовые метрики, отражающие текущее состояние системы. Позволяют отслеживать производительность, загрузку, время ответа. Пример: время отклика HTTP-запросов (`http_server_duration_seconds`).  Инструменты: 
	- **[[notes/10_monitoring/prometheus/basics|Prometheus]]** — стандарт для облачных систем и Kubernetes.
	- **[[notes/10_monitoring/victoriametrics/basics|VictoriaMetrics]]** — быстрая и экономичная альтернатива для хранения огромных объемов метрик

**E - Events**. События, фиксирующие важные события или действия в системе. Пример: развертывание новой версии приложения, регистрация пользователя. Инструменты:
	- **Sentry** — для отслеживания событий об ошибках в реальном времени.
	- **[[notes/10_monitoring/zabbix/basics|Zabbix]]** — традиционный инструмент для мониторинга событий инфраструктуры.

**L - Logs**. Структурированные или текстовые логи, помоющие понять что именно произошло и в каком порядке. Пример: `ERROR OrderService - Failed to create...` Инструменты:
	- **[[elastic-stack|ELK Stack]]** (Elasticsearch, Logstash, Kibana) — самое популярное решение для поиска и анализа логов.
	- **[[loki|Grafana Loki]]** — легковесная система сбора логов, которая тесно интегрируется с метриками.

**T - Traces**. Трассировки, отслеживающие путь запроса через распределенную систему, помогают понять, как компоненты взаимодействуют друг с другом. Пример: трассировка запроса `POST /auth/refresh` показывает последовательность вызовов: API -> OrderService-> InventoryService и тд... Инструменты:
	- **Jaeger** — стандарт для распределенной трассировки в микросервисах.
    - **Tempo** — решение от Grafana Labs для хранения трассировок в объектных хранилищах.


## Методы получения данных (Push, Scrape):
#### Push:

![[Pasted image 20260410171650.png]]

 - Сервис сам отправляет метрики, логи или трейсы в сборщик
 - Используется по умолчанию в логах и трейсах (Loki, Tempo)
 - Требует настройки retry и доставки
#### Scrape:

![[Pasted image 20260410172017.png]]

- Опрос данных, наиболее частоиспользуемый метод
- Сборщик (Prometheus) сам опрашивает endpoints и забирает метрики
- Простая настройка и масштабируемость
- Не подходит для временно живущих сервисов без внешнего адреса
