## Jinja2 (Шаблоны) - Позволяет создавать динамические конфигурационные файлы, подставляя переменные. Используются с модулем `template`

##### Пример шаблона (`roles/nginx/templates/nginx.conf.j2`):
```nginx
server {
    listen {{ http_port }};
    server_name {{ server_hostname }};
    root /var/www/html;
}

```

#### Чтобы конфигурационный файл стал шаблоном ansible (jinja2), ему в конце нужно добавить .j2

##### Процесс выглядит так: Ansible берет шаблон, подставляет в него текущие значения переменных из ваших `vars`, `facts` или `inventory`, и генерирует итоговый файл, который отправляет на сервер.


---

#### Основной синтаксис:

В Jinja2 есть три основных типа конструкций:

- **`{{ ... }}` — Вывод (Variables)**: Печатает значение переменной или результат выражения.
    - _Пример:_ `Hello, {{ username }}`

- **`{% ... %}` — Логика (Statements)**: Используется для циклов, условий и управления структурой.
    - _Пример:_ `{% if status == "active" %} ... {% endif %}`

- **`{# ... #}` — Комментарии**: Не попадают в итоговый файл.

```conf
## Условия (if/else) - Позволяют добавлять строки в файл только при определенных условиях.

server_name {{ domain_name }};
{% if use_ssl %}
listen 443 ssl;
{% else %}
listen 80;
{% endif %}




## Циклы (for) - Незаменимы, когда нужно перечислить список элементов (например, порты в брандмауэре или апстримы в балансировщике).

{% for user in users %}
useradd {{ user.name }}
{% endfor %}




## Фильтры - Позволяют преобразовывать данные «на лету» с помощью символа конвейера `|`.

- `{{ db_password | password_hash('sha512') }}` — хеширование.
- `{{ config_data | to_json }}` — преобразование структуры в JSON.
- `{{ my_list | default(['default_val']) }}` — значение по умолчанию.

```

## Пример использования:

Основной инструмент для работы с шаблонами — модуль **`ansible.builtin.template`**.

**Пример задачи:**  
У вас есть шаблон конфига `templates/motd.j2`:

```j2
Welcome to {{ ansible_hostname }}!
This system is running {{ ansible_distribution }}.
Owner: {{ sysadmin_email }}
```

Плейбук `playbook.yml` для его развертывания:

```yml
- name: Настроить приветственное сообщение (MOTD)
  ansible.builtin.template:
    src: motd.j2        # Путь к шаблону (относительно папки templates/)
    dest: /etc/motd     # Путь на удаленном сервере
    owner: root
    group: root
    mode: '0644'
```
