### Roles - Структурированный способ организации плейбуков. Роль разбивает задачи, переменные, шаблоны и файлы по папкам (задачи -> `tasks/main.yml`, переменные -> `vars/main.yml` и т.д.).

## Создать role

```bash
ansible-galaxy role init [name]
```

## Структура Ansible-проекта:

ansible-project/
├── ansible.cfg          # (1) Конфигурация самого Ansible (опционально)
├── inventories/         # (2) Каталог с инвентарями для разных окружений
│   ├── production/
│   │   ├── hosts.yml    # Инвентарь для продакшена (в YAML или INI)
│   │   └── group_vars/  # Переменные для групп хостов в продакшене
│   │       └── all.yml
│   └── stage/
│       ├── hosts.yml
│       └── group_vars/
│           └── all.yml
├── playbooks/            # (3) Каталог для основных плейбуков
│   ├── site.yml          # Главный плейбук, который запускает всё
│   └── webservers.yml    # Плейбук только для веб-серверов
├── roles/                # (4) Каталог для ролей
│   └── nginx/            # Роль "nginx"
│       ├── tasks/
│       │   └── main.yml  # Основные задачи роли
│       ├── handlers/
│       │   └── main.yml  # Обработчики (например, перезапуск nginx)
│       ├── templates/
│       │   └── nginx.conf.j2  # Шаблоны конфигов (Jinja2)
│       ├── files/
│       │   └── index.html      # Статические файлы
│       ├── vars/
│       │   └── main.yml  # Переменные для роли (высокий приоритет)
│       └── defaults/
│           └── main.yml  # Переменные по умолчанию (низкий приоритет)
├── group_vars/           # (5) Переменные для групп хостов (на верхнем уровне)
│   └── all.yml
├── host_vars/            # (6) Переменные для конкретных хостов
│   └── db-server.yml
└── requirements.yml      # (7) Список внешних ролей/коллекций для скачивания


**Пояснения к структуре:**

1. **`ansible.cfg`** : Файл с настройками. Здесь можно указать путь к инвентарю по умолчанию, пользователя для SSH, отключить проверку ключей хоста (`host_key_checking = False`), увеличить количество параллельных процессов (`forks = 20`) и многое другое.
    
2. **`inventories/`** : Держать инвентари для разных окружений (dev, stage, prod) отдельно — хороший тон. В каждой папке лежит файл со списком хостов и папка `group_vars` с переменными для этого конкретного окружения .
    
3. **`playbooks/`** : Твои главные сценарии. Они, как правило, небольшие и просто вызывают нужные роли.
    
4. **`roles/`** : Сердце проекта. Каждая роль — это логически законченный блок.
    
    - `tasks/main.yml`: Список задач, которые выполняет роль.
        
    - `handlers/main.yml`: Задачи, которые запускаются только по уведомлению из `tasks` (например, перезапустить службу, если конфиг изменился).
        
    - `templates/`: Шаблоны файлов конфигурации с использованием языка шаблонов **Jinja2**. Это позволяет подставлять в конфиги переменные.
        
    - `files/`: Простые файлы, которые нужно просто скопировать на сервер.
        
    - `vars/` и `defaults/`: Переменные. В `defaults` кладут значения по умолчанию, которые можно легко переопределить. В `vars` — внутренние переменные роли, которые менять не предполагается.
    
5. **`group_vars/`** и **`host_vars/`** : Здесь хранятся переменные, которые применяются ко всем хостам из какой-то группы (`group_vars`) или к конкретному хосту (`host_vars`).
    
6. **`requirements.yml`** : Файл для менеджера зависимостей `ansible-galaxy`. Если твой проект использует готовые роли из публичного репозитория (например, с **Ansible Galaxy**), ты перечисляешь их здесь, и одной командой можно скачать их все.

---
### Структура ролей:

- **defaults** - Хранятся переменные с которыми запускается роль (стартовые данные, которые не будут меняться).

-  **Tasks** - Тут, как можно догадаться из названия, хранятся сами таски.

- **Handlers** - Обработчики изменений, если ваша роль исполнялась и что то изменила, то будет вызван handler, как раз та иденпотентность.

- **Tamplates** - Шаблон конфига.

- **Files** - Скрипты, бинарники, любые файлы, которые не нужно шаблонизировать.

- **Tests** - Тесты для ролей.
